AIニュース（2025年12月10日〜12月11日）

本日はOpenAI設立10周年であり、同社はGPT‑5.2のアップデートを発表しました（ブログ、ドキュメント、システムカード）。価格は40%の値上げとなりましたが、全体的に大幅な性能向上が見られます。

GPT‑5.2 Thinkingの各種ベンチマーク比較では、科学的推論や知識労働などで顕著な改善が確認されています。

以前から評価していたGDPValでは、経済的価値のあるタスクで74.1%に到達。「人間の専門家より11倍以上速く、コストは1%未満」という結果から、人間による監督と組み合わせれば専門的業務の支援が可能と示唆されています。

知識労働タスクでのGPTモデル比較では、専門家水準に対する勝率が向上。

先月のGPT‑5.1 Codex MaxではSWE‑Bench Proで苦戦しましたが、5.2 Thinking xhighでは再び改善。

ソフトウェア工学ベンチマークでの精度向上も確認されています。

長文コンテキスト処理も注目点で、MRCR改善が多く報告されています。

ただし完璧ではなく、単語中の文字数を誤るなど単純なミスや、表計算の数値が不正確、視覚機能もGemini 3に劣ると指摘されています。

総じて、今年最後の米国製大型LLMアップデートとしては高評価を得ています。

OpenAI GPT‑5.2の発表：性能、評価、価格、統合

  • GPT‑5.2ファミリー（Instant / Thinking / Pro）
    OpenAIはGPT‑5.2を発表し、知識カットオフを2025年8月31日に更新、コンテキスト拡張、推論負荷の段階的制御を導入しました。難易度の高い推論では、GPT‑5.2 Pro (X‑High)がARC‑AGI‑1で90.5%、ARC‑AGI‑2で54.2%を達成、昨年のo3プレビュー比で約390倍の効率改善。科学・知識分野でも高スコア（例：GPQA‑Diamond 92%以上）。OpenAIは「経済的価値のある業務」を強調し、GDPvalで44職種の70.9%のタスクで人間専門家に勝利または同等と報告（OpenAI、@yanndubs）。API価格は入力$1.75/Mトークン、出力$14/Mトークン、キャッシュ利用で90%割引。

注意点として、コーディングやエージェント性能は混在。SWE‑bench VerifiedではOpus 4.5に劣るケースあり（@scaling01、WebDev Code Arena: 5.2‑high #2）。ツール呼び出しやセキュリティ評価（CVE‑Bench）では5.1 Codex Maxからの改善は限定的（@scaling01）。@polynoamialは、ベンチマーク結果がテスト時の計算量やハーネス設計に敏感であると指摘。

  • 展開とエコシステム
    GPT‑5.2はChatGPTとAPIで利用可能（OpenAI）、Microsoft Copilot（@mustafasuleyman）、VS Code（@code）、Cursor（@cursor_ai）、Perplexity（@perplexity_ai）にも統合。長文推論の改善が報告され（@eliebakouch）、一般推論も強化。ただしLisanBenchでは推論効率でOpus 4.5やGemini 3 Proに劣る（@scaling01）。NVIDIAは5.2を含む先端モデルとのインフラ協力を強調（@nvidia）。

GoogleのInteractions APIとGemini Deep Researchエージェント

  • Interactions API + Deep Research
    Googleはモデルとエージェントを統合的に利用できるInteractions APIを発表。サーバー側状態管理、バックグラウンド実行、MCP対応を備え、初のエージェントGemini Deep Researchを公開（@_philschmid、@GoogleDeepMind）。GoogleはDeepSearchQAをオープンソース化し、深いウェブ検索エージェントの評価を可能に。Deep ResearchはBrowseCompでSOTA、HLEでも高性能（スレッド、ドキュメント）。

実験的には、エージェントのハーネス設計が重要。最小限のオープンソースフレームワーク（Stirrup）がGDPval‑AAでネイティブチャット環境を上回る結果を示し、ツール連携や状態管理、計算予算が成果に大きく影響することを裏付け。

  • 音声
    Googleは低遅延・高品質の新Gemini 2.5 TTSをプレビュー。24言語対応、アクセントや表現力のプロンプト制御が可能（@_philschmid、デモ： @thorwebdev）。
