Perplexity「Computer」：オーケストレーション重視のエージェント製品（マルチモデル、ツール＋環境、使用量ベース課金）

  • Perplexity Computerの発表：PerplexityはComputerを発表しました。これは「調査、設計、コーディング、デプロイ、管理」を1つのインターフェースで行える、ファイル、ツール、メモリ、モデルを統合的に制御するエンドツーエンドシステムとして位置づけられています（発表ツイート、Arav Srinivas）。主な特徴：
    • アクセスと価格：まずはMaxサブスクライバー向けWeb版から提供開始し、その後Pro/Enterpriseへ展開。使用量ベース課金でサブエージェントモデル選択、利用上限、Maxには月1万クレジット＋期間限定ボーナス付与（価格詳細、提供状況、Aravの説明）。
    • アーキテクチャの強調：複数のツイートで「ブレークスルー」は並列・非同期のサブエージェントと、それらにタスクを割り当てるコーディネーターモデルであると説明（調査用、コーディング用、メディア用などの専門モデル）、単一ループ型エージェントではない（Liorの解説、Denis Yarats）。
    • 「Everything is computer」ナラティブ：少人数チームがコーディングエージェントや自動評価・デバッグループを多用して構築したプラットフォームとして発信（Arav、Denis）。
  • エンジニアにとっての意義：Computerは、マルチモデルルーティング、分離環境、永続メモリ、コスト管理など、エージェント作業を単なるチャットではなく分散ワークフローとして扱う、システムレベルのエージェントUXへの具体的な一歩です（Arav、Computerサイト）。

コーディングエージェント：「12月から動き始めた」＋新モデル／ツール（GPT‑5.3‑Codex、Claude Codeエコシステム、Copilot CLI GA）

  • Karpathyの「相転移」発言：Andrej Karpathy氏は、12月以降コーディングエージェントが質的転換を迎えたと主張。脆弱なデモから、長期的タスクを一貫して粘り強く完遂できる段階に移行したと述べています。例として、SSHキー設定→vLLM→モデルDL/ベンチ→サーバーエンドポイント→UI→systemd→レポートまでをほぼ自動で実行（Karpathy）。これは開発ツール界隈の「ソフトウェアが変わりつつある」という感覚と一致（Cursor、snowmaker）。
  • OpenAI GPT‑5.3‑Codexリリースと初期評価：
    • OpenAIはGPT‑5.3‑CodexをAPIで提供開始（snsf）。Clineはサポートを発表し、5.2比で約25%高速化、タスクあたりトークン削減、SWE-Bench Proでの高性能を主張（Cline）。
    • コミュニティのベンチ反応は賛否入り混じり：「IBenchで86%」に驚き（tweet）、「初ベンチ結果まもなく」（kimmonismus）。手法が明確になるまでは参考程度に。
  • Claude Code：製品成熟度＋可観測性＋統合：
    • Claude Codeの「1周年」回顧では基盤的コーディングエージェントとしての位置づけと、コンテキスト長拡張によるメモリ制約への懸念が語られる（swyx）。
    • 実用面ではSlackプラグイン統合（catwu）、LangSmithトレーシングによるルーティング問題のデバッグ（hwchase17、可観測性の課題）。
  • GitHub Copilot CLI GA＋「/research」：
    • Copilot CLIがGAに到達（Evan Boyle）し、リポジトリ全体を対象にGitHubコード検索＋MCP動的取得で深掘り調査する/researchを追加、レポートをgistで共有可能（機能詳細）。
