スキルはMCPの道を辿る！

5.2 Codex や FunctionGemma の小規模なリリースもありましたが、1年後に最も注目されるであろう話題は Claude Skills の継続的な成長です。
この機能は10月に公開され、当初は「Markdownフォルダ」やMCPからの方向転換（現在はLinux Foundationへ移管）として揶揄されました。しかし内部関係者の間では着実に支持を集めています。成長の一例として、Claude Skillsの講演が1日で10万回再生を突破し、これはAIE史上最速であり、2025年で2番目のミリオン再生講演となる可能性があります。

AnthropicのAIエンジニア2名がCode Summitで「エージェントを作るな、スキルを作れ」と書かれたバナーの前で発表している様子

本日の発表内容は以下の通りです：

  • 組織全体でのSkills管理機能
  • MCPと重複しているように見える新しいSkills Directory
  • ベンダーニュートラルな名称「Agent Skills」としてのオープンスタンダード化

これらは一見すると小さな追加機能ですが、全体像としてはSkillsの採用が着実に進んでおり、実際の会話からも多くの人がその重要性を過小評価している可能性があります。

最後にこのようなニュース以外のトレンドを取り上げたのはClaude Codeの時でした。

フロンティアモデルのローンチ：GPT-5.2-Codex、Gemini 3 Flash、オンデバイスGemma派生モデル

  • GPT-5.2-Codex（OpenAI）: OpenAIはGPT-5.2-Codexを「エージェント的コーディング」における最高モデルとして位置付け、ネイティブコンパクション、長文コンテキストの信頼性、ツール呼び出しの改善を強調しました。Codexは有料ChatGPTユーザー向けに提供され、APIは「近日公開予定」です（発表、展開ノート＋サイバー二重用途の枠組み、製品ツイート）。Sam Altmanは現実世界でのセキュリティ影響（Reactの脆弱性開示）を強調し、防御的サイバー機能のための信頼できるアクセスの検討を示唆しました（影響ノート、trusted access）。Greg Brockmanらは「長期的なリファクタリング／移行」や脆弱性発見能力を強調しました（refactor focus、security angle）。
  • Gemini 3 Flashの採用と「速度を製品機能に」: 複数の実務者が、Gemini 3 Flashは速度によって日常的なワークフローを変え、反復サイクルやユーザー行動（定着率／エンゲージメント）に影響を与えると述べています（製品影響の引用、ワークフローの振り返り）。SWE-Bench Verifiedで「FlashがProを上回る」という主張もありますが、ツイート内に統一された評価資料がないため暫定的に扱うべきです（例、別例）。GoogleはFlashをGeminiアプリに統合し、「音声でアプリを構築」する機能として推進しています（Geminiアプリ展開、音声からアプリへの提案）。
  • FunctionGemma（270M）＋T5Gemma 2（エンコーダ・デコーダ）: Google/DeepMindとコミュニティは、小型でオンデバイス／ブラウザでの展開やファインチューニングによる特化を強調しています。FunctionGemmaはテキストのみの関数呼び出し基盤で、ドメイン調整が必要です（デモ＋位置付け、コレクション発表、ファインチューニングガイド）。T5Gemma 2は珍しい現代的なマルチモーダル・多言語エンコーダ–デコーダ系（270M/1B/4B）として紹介されています（リリース）。エコシステムでの即時採用例として、Ollamaによる取得（Ollama）、Unslothノートブック（Unsloth）、MLX対応（MLX例）があります。

エージェント: 「Skills」標準化、ハーネスUX、長時間稼働インフラの現実

  • Agent Skillsが事実上のポータビリティ層に: Anthropicの「Skills」コンセプトは、指示／スクリプト／リソースを含む相互運用可能なパッケージ形式として広まりつつあり、ツールチェーン採用の強い兆候があります。VS Codeがオープンスタンダードをサポート（@code）、コミュニティではSkillsをMCPの標準化の流れに類似と捉える声もあります（@omarsar0、@alexalbert__）。Artificial AnalysisのStirrupはディレクトリからのSkills読み込み（Markdown優先）を追加し、Claude Code/Codexスタイルでの再利用を狙っています（Stirrup対応）。
  • ハーネスはエージェントUXの「配布」: 複数のツイートで、エージェント性能はモデル品質だけでなく「ハーネスの選択」（並列ツール呼び出し、メモリUX、コンパクション方針、サブエージェントのオーケストレーションビュー）といった製品面に依存すると述べられています（harness take、agent blocks mental model）。
  • インフラのミスマッチ: サーバーレス vs エージェントループ: エージェントシステムには永続的で長時間稼働する実行と状態管理が必要であり、「サーバーレス」パターンでは多段ループに脆弱なネットワーク回避策が必要になるという指摘があります（批判）。これは、バックグラウンドエージェントに適したオーケストレーション基盤（例: Temporal）への関心の再燃と一致します（@corbtt）。
  • Claude Codeの機能拡張（ウェブブラウジング）: Claude Codeがウェブ閲覧可能になり、Xフィードをフィルタして非同期に報告する「監視」エージェントが構築可能になったとユーザーが報告しています（構築例）。
