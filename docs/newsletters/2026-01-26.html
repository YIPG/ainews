豊かな生成UIこそがすべてです。

3か月前、OpenAIがChatGPT AppsとApps SDKをDev Day 2025で試験的に発表した後、Anthropicは独立したMCP UIプロジェクトを正式に吸収しました。そしてOpenAI、Block、VS Code、Antigravity、JetBrains、AWSなどと協力し、以下をリリースしました。

  • MCP Apps仕様
  • Claude.aiでの公式サポート

ChatGPT Appsは発表以来、世界を席巻しているとは言い難いですが、アプリケーションが豊かなUIを返すための標準フォーマットの必要性は否定できません。重要なプレイヤー全員がMCP Appsを承認したことで、オープンソースのサポートやアプリケーション間の相互運用が可能となる豊かなエコシステムの基盤が整いました。これにより、将来的には毎月20ドルのサブスクリプションがクレジットカードに積み重なる問題を解決できるかもしれません。

エージェントオーケストレーション、RLM、そしてUXパターンとしての「Clawdbot/Clawd」

  • NVIDIA ToolOrchestra + Orchestrator-8B：NVIDIAのToolOrchestraは、エージェントシステムを「小さな指揮者モデル」として位置づけ、推論とツール呼び出し、大規模な専門モデル（検索、コード実行、専門LLM、汎用LLM）を交互に行います。8Bオーケストレーターが委任によってフロンティアレベルの成果を低コストで達成できると主張しており、現実的なツール呼び出しのロールアウトとマルチターンタスクを用いたスケーラブルRLでエンドツーエンド学習されています（概要、リンク）。技術的示唆としては、現実的なツール呼び出しのロールアウトで訓練できるなら、「コントローラの規模」よりもポリシー品質＋ツール／モデルルーティングが重要ということです。
  • RLM / 再帰優先のエージェントスタック：複数の投稿がRecursive Language Model（RLM）パターンに収束しています。これはファイルやコンテキストを参照渡しし、必要最小限のスライスを反復的に取得する（shell/grep/AST）もので、ReActのようにすべてをコンテキストに詰め込む方法とは異なります。Dan Bは、ファイル参照と@file展開の違いを意図的なコンテキスト管理として説明しています（スレッド）。Daytonaは、サブエージェントごとのサンドボックスによる「無限の再帰深度」としてRLMを位置づけています（ガイド、統合）。
  • 「Clawd/Clawdbot」ミーム→製品シグナル：データセットには「Clawdbot」の波（Mac miniジョーク付き）が多く含まれていますが、技術的に重要なのは成果重視のアシスタントUXとコンテキスト／ツール統合の緊密さです。Kimmonismusはこれを「より多くのチャット」から「より多くの成果」へのシフトと呼び、既存企業が追随を急ぐだろうと示唆しています（ツイート）。一方でクラウド優先（ローカルMac mini不要）という反論もあります（MiniMaxの返信）。また、「強力モード」が存在するとすぐにセキュリティ反発が起こり、ブラウザ／デスクトップエージェントではプロンプトインジェクションが依然としてシステムレベルの障害となっています（ジレンマ、フォローアップ、Miesslerの警告）。

推論モデルのリリースと評価動向（Qwen、Tencent、ARCなど）

  • Alibaba Qwen3-Max-Thinking：AlibabaはQwen3-Max-Thinkingを、検索／メモリ／コードインタプリタなどの適応的ツール利用とテスト時スケーリング／自己反省を備えた推論＋エージェントモデルとして発表しました。数学やエージェント検索で高い評価を得ており（例：HMMT Febで98.0、HLEで49.8）、発表後すぐにLM ArenaやYuppで評価に投入されています（Arena、Yupp）。コミュニティは、検索ツールを使った評価体制がSOTAモデルを上回る可能性を指摘しています（コメント）。
  • Tencent HunyuanImage 3.0-Instruct（画像編集）：Tencentは画像編集に特化したマルチモーダルモデルを発表しました。80B MoE（13Bアクティブ）、ネイティブCoT、MixGRPOアルゴリズムを採用し、非対象領域を保持した精密な編集や複数画像の融合に対応します（発表）。LM Arenaでは画像編集ランキングで7位に入りました（Arena）。
  • ARC-AGIのコスト／性能ハック：「Recursive Self-Aggregation（RSA）＋Gemini 3 Flash」で、Gemini Deep Thinkの約1/10のコストでARC-AGI-2で59.31%を達成したという報告があります（ツイート）。これは、集約、再帰、枝刈りなどのメタ推論戦略が基盤モデル選択と同等に重要になっていることを示しています。
  • オープンモデルの評価参加：Molmo 2（Apache 2.0）がArenaに新たなオープンモデルとして登場しました（Arena）。また、Hugging Face Inference EndpointではGLM-4.7-Flashをllama.cpp経由で提供し、低価格で24kコンテキストを実現しています（ngxson）。これは高速なオープンウェイト推論のコモディティ化を示しています。
