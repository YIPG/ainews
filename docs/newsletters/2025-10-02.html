AI Twitterまとめ

動画生成：Sora 2、Kling 2.5 Turbo、Google「Nano Banana」GA

  • Kling 2.5 Turbo（Text/Image→Video） : Klingの最新モデルが、Artificial Analysis Video Arenaでテキスト→動画、画像→動画の両部門でトップとなり、Hailuo 02 Pro、Google Veo 3、Luma Ray 3を僅差で上回りました。最大1080pで5秒/10秒のクリップを生成可能です。コストはFAL APIで約$4.20/分（Hailuo 02 Proは$4.90、Seedance 1.0は$7.32）で、アプリクレジット利用のUltraプランでは1本あたり約15¢。モデル比較と価格は @ArtificialAnlys のスレッドや @Kling_ai の発表をご参照ください。
  • OpenAI Sora 2：能力 vs 正確性 : 実際の利用では高度な指示追従やアプリ内リミックスが可能ですが、物理的整合性の欠如やマーケティング的演出が指摘されています。デモまとめは @altryne、物理的忠実性より「迎合的」な傾向の批評は @teortaxesTex、Veo 3が物理シナリオで優れる事例は @fofrAI、全体的な評価は @Tim_Dettmers を参照。
  • Google Gemini 2.5 Flash Image（「Nano Banana」）GA : 本番利用可能となり、10種のアスペクト比、複数画像のブレンド、画像のみ出力に対応。価格はGemini API（AI Studio + Vertex）で$0.039/画像。発表は @sundarpichai、@GoogleAIStudio、@OfficialLoganK など。Cartwheelの新モーションパイプライン統合例は @andrew_n_carr、Google開発者アカウントのデモは @googleaidevs を参照。
  • エコシステム : Synthesia 3.0が「ビデオエージェント」と新ワークフローを追加 @synthesiaIO。

オープンウェイトモデルリリース：IBM Granite 4.0とQwenアップデート

  • IBM Granite 4.0（Apache 2.0、ハイブリッドMamba/Transformer） : IBMの新ファミリーは、標準アテンション層を少数、Mamba層を多数組み合わせ、メモリ削減と精度維持を両立。サイズはGranite 4.0 H Small（MoE 32B/9B稼働）、H Tiny（7B/1B）、H Micro（3B/3B）、3B密結合Microなど。128Kコンテキスト、Apache 2.0、トークン効率良好。Artificial AnalysisによるとH SmallはIntelligence Indexで23（Gemma 3 27Bの22を上回り、Mistral Small 3.2の29には及ばず）。Microは16でGemma 3 4Bの15を上回る。HuggingFaceやReplicateで利用可能（H Smallは入出力100万トークンあたり$0.06/$0.25）。ベンチマークは @ArtificialAnlys、Ollamaは各種モデルの実行イメージを提供 @ollama。LM Arenaにも追加 @arena。
  • Qwenアップデート : QwenモデルがTinkerのファインチューニングAPIに対応 @wzhao_nlp。Qwen-Image-2509は一貫性向上 @Alibaba_Qwen、Qwen3 VL 235Bは一部のビジョンタスクで低コスト高性能と報告 @scaling01。

ファインチューニングとシステム：Tinker、rank-1 LoRA、MoE対応、推論高速化

  • Tinker：柔軟なファインチューニングAPIとLoRA共有 : Thinking MachinesのTinkerは、CPUのみのトレーニングループを分散GPU上でそのまま実行可能にし、アルゴリズムや損失関数の制御を保持しつつ、スケジューリングやリソース管理を自動化。LlamaやQwenなどのオープンモデルや大規模MoEに対応。概要は @TheTuringPost、リリースノートは @Smol_AI。
  • rank=1 LoRA : 複数の再現実験で、rank-1 LoRAが推論タスクでフルファインチューニングと同等の品質を達成しつつVRAMを約43%節約、より大きなモデルでのRLを可能に。結果とコードは @zzlccc、Colab例は @ben_burtenshaw。
  • MoEトレーニングとインフラ : Prime-RLがQwen3 A3-30BやGLMシリーズなどでMoE対応、Torch Compile互換を保ちながらHFエコシステムと互換性維持。推論では @vikhyatk が1.3〜20倍高速化を報告。MI300X VMは$1.99/GPU/時でオンデマンド提供 @HotAisle。

RLと推論：学習中探索、探索拡大、潜在CoT、推論前倒し

  • 学習時探索と効率的探索 : DeepSearchはMCTSを学習ループに組み込み、AIME/AMCで62.95%達成（330GPU時間でNemotronベースラインを上回る）。BroRLは例ごとのロールアウト数を数百に増やし、学習ステップ増加だけでは飽和する問題を克服。
  • アーキテクチャと学習機構 : 新しい潜在CoT「thoughtbubbles」は入力適応型の潜在トークンを挿入し、CoTラベルなしで計算資源を増加させ、パープレキシティと計算効率を改善。NVIDIAの「Front-Loading Reasoning」は事前学習時に推論を注入することで持続的な性能向上を確認。

エージェントとツールチェーン：CLI＋セマンティック検索、Notebook MCP、ブラウザ、CLI

  • CLIエージェント＋セマンティック検索 : LlamaIndexのSemToolsベンチマークでは、セマンティック検索を組み合わせたエージェントがCLIのみのエージェントより包括的な回答を生成。
  • MCP経由でノートブック実行 : GoodfireがScribeをオープンソース化、エージェントがノートブックセルを実行しJupyter出力を受け取れるように。
  • 「AIブラウザ」と評価者 : PerplexityのCometが全世界GA、Comet Plusも開始。Yuppの「Help Me Choose」は第三のモデルが候補回答を批評し合う仕組みを採用。

リーダーボードと実運用コードエージェント指標

  • Claude Sonnet 4.5がLM Arenaで1位タイ : Claude Opus 4.1と並びトップに。コーディングやクリエイティブライティングでも高評価。
