Claude Code + Hugging Face SkillsによるオープンLLMトレーニングの自動化

  • 「ワンライン」で完結するファインチューニングパイプライン：Hugging FaceはClaude Code用の「skill」を公開しました。これにより、自然言語で「Fine-tune Qwen3-0.6B on open-r1/codeforces-cots」と指定するだけで、データセット検証、GPUタイプ選択、HF Jobsの起動、進捗監視、Hubへのチェックポイントやモデル公開までを自動で実行できます。SFT、DPO、GRPOに対応し、約0.5B〜70Bのモデルをサポート、GGUFエクスポートや多段パイプラインも可能です。小規模実行では約$0.30のコストとの報告もあります。詳細は@LiorOnAI、HFブログリンクはこちら、さらに詳しい解説は@akshay_pachaarをご覧ください。

新しいマルチモーダルモデル：ZhipuのGLM‑4.6VとJina‑VLM

  • Zhipu AIのGLM‑4.6VとGLM‑4.6V‑Flash：128kコンテキストとネイティブなマルチモーダル関数呼び出し機能を備えた新しいVLMです。GLM‑4.6Vは総パラメータ106BのMoEで、約12Bがアクティブ。Flashはレイテンシやローカル展開に最適化された9Bの密モデルです。価格（API、100万トークンあたり）：入力$0.6／出力$0.9、Flashは無料。重みはHFで公開、vLLMは初日からレシピを提供、MLX‑VLMもサポートを追加、複数のプラットフォームが画像引数付きのツール呼び出しを統合しました（OCR不要）。詳細は@Zai_org、HF重みは@_akhaliq、MoE詳細は@ben_burtenshaw、vLLMレシピは@vllm_project、MLX‑VLMは@Prince_Canuma、ツール呼び出し統合は@ZenMuxAIをご覧ください。初期コメントでは106Bと9Bの性能差が小さいとの指摘もあります（@eliebakouch）。
  • Jina‑VLM (2B)：図表、チャート、シーンテキスト、文書に特化したコンパクトな多言語VLMです。Jinaによれば、8つのVQAベンチマーク平均72.3で、MMMB（78.8）やMultilingual MMBench（74.3）で最高性能を達成。詳細は@JinaAI_およびフォローアップ1 2をご覧ください。

NeurIPS 2025研究動向：新しいシーケンスアーキテクチャ、解釈可能性、形式手法

  • ポストTransformerバックボーン（GoogleのMirasフレームワーク）：Googleの論文は、TransformerやRNNを連想記憶システムとして再定義し、「忘却」を保持の正則化として捉え、Moneta、Yaad、Memoraを提案。LM、推論、長文コンテキストスケーリング、needle-in-a-haystack検索でTransformerやMamba2、DeltaNet、ハイブリッドを上回り、長文検索で最大約20%の改善を報告。概要は@TheTuringPost、arXivリンクはこちら。
  • 形式手法と最先端数学の融合：AxiomProverは、Leanを用いた自律システムがPutnam 2025の12問中9問を試験終了後数時間以内に解いたと報告。昨年のスコアボードを上回る性能で、検証可能性と形式／非形式のハイブリッドパイプラインを強調。詳細は@axiommathai、@TheTuringPostによる「AI数学者」議論をご覧ください。
  • 大規模機械的解釈可能性と実用性向上：解釈可能性ワークショップは大盛況で、Chris Olahが「解釈可能性の振り返り」について講演（@NeelNanda5）。分野の声として、単一モデルのニューロン事例よりも、スケーラブルで汎用的なツールの必要性が強調されました（@sarahookr）。
  • ツール豊富な現実的環境でのエージェント評価：MEMTRACKはSlack、Linear、gitタイムラインを備えた「職場」環境で長期的な記憶／状態追跡を評価。最高スコアはGPT‑5の約60%で、改善余地が大きいことを示しました（@rebeccatqian）。

エージェントの実践：評価、信頼性、知識の基盤

  • Deep Agents評価パターンと結果：LangChainは長期稼働エージェントの評価パターン（計画、FS、サブエージェント、プロンプト）を公開し、Terminal Bench 2.0で平均約42.65%のスコアを記録。また、動的コンテキスト圧縮トリガー（例：ウィンドウ85%で要約、10%保持）や、LangSmithによる観測・評価・展開の動画シリーズも提供。詳細は@LangChainAI、コンテキスト圧縮は@sydneyrunkle、LangSmithシリーズは1 2をご覧ください。
  • 消費者向けの知識基盤は依然弱い：ACEベンチマークは買い物、食品、ゲーム、DIYなどの日常タスクを対象に、ウェブソースの動的チェックを行います。最高スコアはGPT‑5 High=56.1%、o3 Pro=55.2%、Shoppingは45.4%が上限。一部モデルはリンク精度でマイナススコア（例：Gemini 3 Pro -54%「Provides link[s]」）。詳細は@omarsar0。
  • ツールとワークフロー：Dexter 2.0（オープンソース、LangChainベース）は計画／自己検証を備えた自律的金融調査を対象（#demo）。AI21 Maestroは多段階計画、組み込み検証、独自RAG、実行グラフを備えたオーケストレーションを提供（@AI21Labs）。DSPyのGEPAは新タスク統合で大きな性能向上を示し（12.5%→62.5%）、@DSPyOSSが報告。
  • 「LLMは新しい知識を生成できない」論への反論：@jeremyphowardは、ツールを補助に使ったLLMが未記録の結果を導き出す事例を示し、エージェントが相互作用を通じて新しい知識を生成できると主張しています。
