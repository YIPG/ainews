AnthropicのClaude Opus 4.7とClaude Designの展開

  • Claude DesignがAnthropic初のデザイン／プロトタイピングツールとして登場 : @claudeai が発表した Claude Design は、自然言語の指示からプロトタイプ、スライド、ワンページ資料を生成するリサーチプレビュー版ツールで、Claude Opus 4.7 によって動作します。この発表により、Anthropicはチャットやコーディングからデザインツール領域へ進出したと見られ、複数の観測者がこれを Figma/Lovable/Bolt/v0 への直接的な挑戦と評しました（@Yuchenj_UW、@kimmonismus、@skirano）。発表後、Figma株価の急落も話題となり、@TheRundownAI によって製品詳細が共有されました。機能はインライン修正、スライダー調整、Canva/PPTX/PDF/HTML へのエクスポート、実装用の Claude Code への引き渡しなどです。
  • Opus 4.7は全体的に強化されたが、ローンチは混乱気味 : サードパーティのベンチマーク投稿は概ね好意的でした。@arena は Opus 4.7 をCode Arenaで1位に評価し、Opus 4.6より+37ポイント、他社モデルを上回る結果を報告。同アカウントはText Arenaでも総合1位とし、コーディングや科学分野での勝利を記録しました（詳細）。@ArtificialAnlys は知能指数評価で Opus 4.7 57.3 、Gemini 3.1 Pro 57.2 、GPT-5.4 56.8 とほぼ三つ巴で、GDPval-AA（エージェント性能評価）ではOpus 4.7が首位と報告。出力トークン数はOpus 4.6より約35%減少し、タスク予算機能が導入され、拡張思考は完全に廃止され適応型推論に置き換えられました。ただし初日のユーザー体験は賛否両論で、@VictorTaelin は性能低下やコンテキスト失敗を報告、@emollick は翌日には適応思考の改善があったと述べ、@alexalbert__ は初期バグの多くが修正されたと確認しました。@theo はDesign機能の安定性に不満を示し、同アカウントは安全性問題も指摘しました（詳細）。
  • コスト／効率の議論が品質と同等に重要に : @scaling01 は一部のML問題で従来の高性能モデルに比べ約10倍少ないトークンで同等性能を維持したと主張。@ArtificialAnlys はOpus 4.7をテキストとコード両方で価格性能のパレート最適に位置付けました。絶対的な首位評価は一致せず、@scaling01 は Gemini 3.1 Pro や GPT-5.4 にLiveBenchで劣ると指摘しましたが、総じてエージェント性能と効率が大幅に改善されたとの見方が多いです。

コンピュータ利用、コーディングエージェント、ハーネス設計

  • コンピュータ利用UXが主流製品カテゴリに : OpenAIのCodexデスクトップ／コンピュータ利用機能更新は実務者から強い反応を得ました。@reach_vb は サブエージェント＋コンピュータ利用 が実用感覚としてAGIに近いと述べ、@kr0der、@HamelHusain、@mattrickard、@matvelloso はCodex Computer Useが派手さだけでなく高速で、Slack、ブラウザフロー、任意のデスクトップアプリを操作でき、企業のレガシーソフトに対応する初の実用的プラットフォームとなる可能性を強調しました。@gdb はCodexを完全なエージェント型IDEと位置付けています。
  • 分野は「シンプルなハーネス、強力な評価、モデル非依存のスキャフォールド」に収束 : 複数の投稿が、信頼性向上は巨大モデル追求よりハーネス設計から得られると主張。@AsfiShaheen は金融分析パイプライン（ルーター／レーン／アナリスト）を紹介し、バグの多くは指示やインターフェースの問題だと指摘。@AymericRoucher はClaude Codeのリークハーネスから同様の教訓を抽出。@raw_works はさらに顕著な例を示し、Qwen3-8B がLongCoT-Miniでdspy.RLM使用時に33/507、バニラでは0/507で、スキャフォールドが性能向上の全てを担ったと述べました。LangChainはこれらパターンを製品に組み込み、@sydneyrunkle は deepagents deploy にサブエージェント対応 を追加、@whoiskatrin はAgents SDKにメモリプリミティブを導入しました。
  • オープンソースエージェントスタックの拡大 : Hermes Agentが注目を集め、@GitTrend0x は派生プロジェクト（Hermes Atlas、Hermes-Wiki、HUD、制御ダッシュボード）を紹介。@ollama は ollama launch hermes によるネイティブHermes対応を提供し、@NousResearch が拡散。NousとKimiは$25k Hermes Agent Creative Hackathonを開始し、コーディング／生産性からクリエイティブエージェントワークフローへ拡張を示しました。
