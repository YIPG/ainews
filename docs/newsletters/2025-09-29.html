AI Twitterまとめ

DeepSeek V3.2-Exp：Sparse Attention、値下げ、オープンカーネル

  • DeepSeek Sparse Attention（DSA）を採用したV3.2-Expモデルを発表。V3.1-Terminusに学習済みSparse Attentionを後付けし、長文コンテキストのコストを品質低下なしで削減。過去トークンをクエリごとにスコアリングし、上位k位置のみ全Attentionを実行することで計算量をO[L²]からO[Lk]に変更。2段階の継続事前学習を実施。モデル、技術レポート、カーネルを公開し、API価格を50%以上引き下げ。詳細は@deepseek_ai。
  • エコシステムとコンパイラ：vLLMがDSA対応レシピとH200/B200ビルドを提供。DeepSeekのカーネルはTileLang/CUDAで提供され、TileLangは約80行でFlashMLAの95%性能を達成。
  • ポストトレーニングレシピ：DeepSeekは専門モデル（数学、競技プログラミング、推論、エージェントコーディング、検索）に対してGRPOとルーブリック/一貫性報酬でRLを行い、最終チェックポイントに蒸留。

Anthropic Claude Sonnet 4.5：コーディング・エージェント性能向上と初の解釈可能性監査

  • コーディングとエージェント性能のSOTA：SWE-Bench Verifiedで新記録を達成し、OSWorldや長時間自律コーディングで大幅な性能向上。価格は据え置き。
  • アラインメントと解釈可能性：詳細なsystem cardを公開し、評価環境認識などの信号を解釈可能性研究で発見。事前展開時にホワイトボックス監査を実施。
  • ツールと統合：Claude Code v2でチェックポイントやUX改善、VS Code拡張を提供。Claude Agent SDKとして一般エージェント向けに再設計。Cursor、Perplexity、OpenRouterなどで利用可能。

LLMのRL：GRPO vs PPO vs REINFORCE、LoRAが多くの設定で全量FTに匹敵

  • GRPO議論：GRPOはREINFORCEの変種であり、性能差はデータレシピやクレジット割当の方が影響大。
  • LoRAの有効性：LoRAは多くのRL後処理で全量FTに匹敵する性能を示す。
  • データボトルネック論争：LLMは人間生成データに依存しており、AGIは計算資源依存になる可能性。

エージェント型コマースとプラットフォーム更新

  • OpenAI Instant Checkout + ACP：ChatGPTで直接購入が可能に。Stripeと共同開発したオープン標準を採用。
  • 安全性とガバナンス：OpenAIがペアレンタルコントロールを導入。Anthropicはカリフォルニア州SB53を支持。
