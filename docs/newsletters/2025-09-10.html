Oracleおめでとうございます！

本日は Anthropic MCP Registry の公式発表や ChatGPT Developer Mode、Claude の新しいVM、Mistralの大型資金調達 なども取り上げる予定でしたが、最大のニュースは Oracle の OCI部門 です。予想を大きく上回り、予約売上高が前年比+359%の4,550億ドル に達し、2030年までにクラウド収益1,440億ドルを見込むガイダンスを発表しました（参考：OCIは現在180億ドル、AWSは1,120億ドル、Azureは750億ドル）。株価は時価総額で2,500億ドル以上上昇し、1兆ドルクラブ入り目前。Larry Ellison氏は世界一の富豪となり、歴史的な快進撃となりました。

The Wall Street Journal の報道によると、この予約売上の多くはOpenAIによるものであり、これはMicrosoftとの数か月にわたる緊張関係の結果でもあるとされています。

AI Twitter Recap

ツール利用と重み更新インフラ向け高速RL（Kimi checkpoint-engine, RLFactory, TRL）

  • Kimi’s checkpoint-engine（オープンソース）：Moonshot AIが、大規模推論環境でモデルの重み更新をインプレースで行える軽量ミドルウェアを公開しました。1兆パラメータモデルを数千GPUに約20秒で更新可能で、ブロードキャスト（同期）とP2P（動的）モードをサポート。H2D転送、ブロードキャスト、リロードを重ね合わせ、vLLMと統合。詳細は @Kimi_Moonshot、@vllm_project のベストプラクティス、@vllm_project の最適化手順スレッドを参照。背景として、@ZhihuFrontier による詳細解説では、RDMA Writeを用いたノード間2秒同期（Qwen3‑235B, BF16→FP8）をディスクI/OやCPUを介さず実現する方法が紹介されています。
  • RLFactory（LLMツール用プラグアンドプレイRL）：非同期ツール呼び出し（6.8倍スループット）、訓練と環境の分離、柔軟な報酬設計、小規模モデルが大規模ベースラインを上回る事例（Qwen3‑4B > Qwen2.5‑7B）を示すフレームワーク。詳細は @arankomatsuzaki とリポジトリ。
  • TRL v0.23：任意のコンテキスト長での学習を可能にするContext Parallelismを導入。長文コンテキストのSFT/RLに有用。詳細は @QGallouedec。
  • Prime Intellect RL stack：軽量RFTがprime-rl、検証器、Environments Hubと統合され、オープン開発者向けのフルスタックSOTA RLインフラへ拡張中。発表。

決定論的かつスケーラブルな推論/学習（vLLM determinism, BackendBench, dynamic quant, HierMoE）

  • LLM推論の非決定性解消：Thinking Machines Labが研究ブログ「Connectionism」を開始し、浮動小数演算、カーネル、キャッシュ、サンプリング整合性などを含む決定論的推論パイプラインの実践ガイドを公開。Qwen向けにvLLMを決定論化する最小パッチも提供。発表、@cHHillee、@vllm_project の事例参照。PyTorch nightlyはBlackwell向けCUDA 13ホイールを提供開始（@StasBekman）。
  • BackendBench（Meta主導）：PyTorchバックエンドのオペレーター対応を評価するベンチマーク。Prime Intellect Environments Hubで比較・議論が容易に。詳細。
  • 動的量子化ノート（DeepSeek V3.1）：@danielhanchen によると、“thinking”モードでは低ビットでも高精度を維持し、3ビットでFPに近い精度を達成。attn_k_bを8ビットに保つと4ビット比で+2%精度向上。
  • HierMoE（MoE学習システム）：階層間のトークン重複排除とエキスパート入れ替えによりAll-to-All通信を1.55–3.32倍、エンドツーエンドを1.18–1.27倍高速化。top‑kルーティングが多いほど効果増。概要。
