Inference Systems、vLLM 0.20、そしてDeepSeek V4を巡るハードウェア／カーネル競争

  • vLLMの最新リリースはメモリとMoE推論効率に重点 : vLLM v0.20.0 は、TurboQuant 2-bit KV cache による 4倍のKV容量、SM90+でのMLAプリフィルにFA4再有効化、新しいvLLM IR基盤、融合RMSNormによる2.1%のエンドツーエンド遅延改善、さらにDeepSeek V4 MegaMoE on Blackwell、Jetson Thor、ROCm、Intel XPU、GB200/Grace-Blackwellのセットアップ簡易化などのサポート更新を含みます。同時に、SemiAnalysis はB200/B300/H200/GB200分離構成でのDeepSeek V4 Pro推論結果を紹介し、B300がこのワークロードでH200より最大8倍高速であると主張。さらにvLLM 0.20でのDeepGEMM MegaMoEベンチマーク予定を示し、これはEP dispatch＋EP combine＋GEMMs＋SwiGLUを単一メガカーネルに融合します。
  • エコシステムは新しいオープンモデルへのDay-0対応に収束 : vLLMはPoolsideのLaguna XS.2にDay-0対応、Ling-2.6-flash、NVIDIAのNemotron 3 Nano Omniにも対応。vLLM外では、推論のトレードオフに関する投稿が増加。Jeremy HowardはDeepSeek V4のprefill対応を、多くのプロバイダが削除している機能として指摘。Maharshiは動的活性化量子化のオーバーヘッドを指摘し、キャリブレーションコストがあっても静的量子化が推論速度で勝ることが多いと主張。またteortaxesTexはTileKernelsによるCUDAロックインからの構造的脱却を指摘し、モデルベンダーがNVIDIA専用展開ではなく異種・国内アクセラレータ向け最適化を進める可能性を示唆。

オープンモデルリリース: Poolside Laguna XS.2、NVIDIA Nemotron 3 Nano Omni、TRELLIS.2

  • Poolsideは初の公開モデルを発表、展開しやすいオープンウェイトのコーディングモデル : @poolsideaiはLaguna XS.2を発表、33B総パラメータ／3BアクティブMoEのコーディングモデルで、完全に社内で学習、Apache 2.0ライセンスで公開、単一GPUで動作可能。Poolsideの広範なリリースにはLaguna M.1やエージェントハーネスも含まれ、データ、学習インフラ、RL、推論スタックを自社構築。コミュニティのまとめではAymeric Roucherが225B/23Bアクティブと33B/3Bアクティブの2つのコーディングモデル、ハイブリッドアテンション、FP8 KV cache、Qwen-3.5に近い性能を説明。Ollamaは即日提供。
  • NVIDIAのNemotron 3 Nano Omniはインフラネイティブな大型モデルの最大の発表 : @NVIDIAAIがNemotron 3 Nano Omniを紹介、オープンな30B / A3BマルチモーダルMoE、256Kコンテキスト、テキスト・画像・動画・音声・文書に対応。即日でOpenRouter、LM Studio、Ollamaなど多数が提供。仕様としてPiotr ŻelaskoはParakeetエンコーダによる音声理解、現状英語のみ、Open ASRリーダーボードで5.95% WERを報告。複数ホストが同等のオープンオムニモデルに比べ約9倍のスループットと説明。
  • その他の注目モデル／論文 : MicrosoftのTRELLIS.2はオープンソースの4B画像→3Dモデルで最大1536³のPBRテクスチャ資産を生成、ネイティブ3D VAEで16倍空間圧縮。ワールドモデル分野ではWorld-R1が既存動画モデルがすでに3D構造を符号化しており、RLで「目覚めさせる」ことが可能と主張、追加データや構造変更不要。

エージェント、ローカルファーストツール、プロダクションオーケストレーション

  • エージェント構築はデモからプロダクション基盤へ移行 : MistralはWorkflowsをパブリックプレビューで開始、企業AIプロセスを耐久性・可観測性・耐障害性のあるプロダクションシステムに変えるオーケストレーション層。Sydney Runkleは耐久実行を長時間稼働エージェントの必須要件と説明。threepointoneは永続性・ストリーミング・再開を備えたサブエージェント／ツールとしてのエージェント開発を紹介。
  • ローカル／オフラインエージェントが実現可能なワークフローに : Tekniumは「完全オフラインエージェントは可能」と主張、Niels RoggeはPi＋ローカルモデルでデスクトップ整理をデモ、Google Gemmaはローカルコーディングエージェントのチュートリアルを共有。Hugging Faceのローカル推進も採用数で顕著に。AmmaarはGemma 4をMLXで完全オンデバイス動作するアプリを公開、KimmonismusはSigmaを紹介、オープンモデルを用いたプライベートブラウザベースのローカルエージェント概念。
  • Hermesや関連エージェントハーネスが実務で普及 : HermesがOpenClawを指示追従や実務ワークフローで上回る事例が複数報告。SecretArjun、somewheresy、Telegram経由のHermes利用、医療文献抽出など。研究エージェントではHugging FaceのML InternがSpacesで話題となり、Trackio統合によるネイティブメトリックログを追加。

注目すべきベンチマーク・評価・研究結果

  • モデルベンチマークは断片的だが注目信号あり : EpochはGPT-5.5 ProがEpoch Capabilities Indexで159に到達、FrontierMathで新記録—Tier1–3で52%、Tier4で40%、Tier4の未解決問題2つを初解決。Greg KamradtはGPT-5.5とOpus 4.7のARC-AGI-3テスト完了を報告。
  • より現実的なエージェント・エンジニアリング行動を測る新ベンチマーク : LysandreはTransformerをエージェント向けにするベンチマークを発表、VibeBenchは1,000人の資格あるソフトウェアエンジニアによる主観評価を提案。LlamaIndexのParseBenchはOCRベンチマークが意味を変える取り消し線や上付き文字などを見落としていると指摘。
  • 具体的な工学的示唆を持つ研究メモ : RosinalityはDeepSpeedとOpenRLHFのバグがSFT性能を低下させると指摘、Arjun KocherはDeepSeek-V4論文のCompressed Sparse Attentionを忠実実装、che_shr_catは単一ブロックTransformerがExtreme Sudokuを解くには明示的スクラッチパッドと逆ルーティング初期化が必要と示す、Keller JordanはMuonやAdamW比較用の軽量Modded-NanoGPT最適化ベンチマークを公開。

AIガバナンスと防衛: Googleの国防総省契約に社内から強い反発

  • 最も論争を呼んだ政策ニュースはGoogleの機密国防総省AI契約 : KimmonismusはGoogleが機密業務や「合法的な政府目的」へのAI利用を許可する契約を締結したと報告、安全フィルタ変更要求を政府が可能にする条項、監視や自律兵器への非拘束的制限のみを含む。Google/DeepMind内部からBlackHCが「恥ずべき」と批判、事前告知や議論なしと発言。
  • 反応はフロンティアラボのレッドラインの違いを鮮明化 : S. Ó hÉigeartaighはGoogle DeepMindもOpenAIと同じ基準で精査すべきと主張、TurnTroutはGoogleの条件がOpenAIの制限より弱いと指摘。この件はAnthropicの対照的姿勢も強調。エンジニアにとっては政治よりもプラットフォームガバナンス—安全ポリシー、展開制御、契約文言が製品の一部になりつつあることが重要です。
