AI Twitter Recap

Kimi Linear (KDA)、Minimax M2、そしてリニアアテンション戦争

  • Kimi Linear (KDA) が初日からインフラと長文コンテキスト性能を搭載して登場 : Moonshot AIがKimi Linear技術レポートとチェックポイントを公開しました。これはKimi Delta Attention (KDA)とMLAを組み合わせたハイブリッド構造（約3:1のKDA:MLA比率）で、最適化されたKDA CUDAカーネルをオープンソース化し、vLLMに初日から統合しています。報告された成果は、KVキャッシュ最大75%削減、最大6倍のデコードスループット（1Mコンテキストで6.3倍TPOT）、長文コンテキストやRL長文推論タスクで全アテンションと同等以上の品質などです。vLLMではRULER 128k = 84.3、約4倍の速度向上を確認。さらにMLA層では位置エンコーディングなし（"NoPE"）でも長文コンテキストが有効であることを報告しています。リンク: @Kimi_Moonshot、@scaling01、vLLM、@uniartisan。
  • Minimax M2のフルアテンションへの転換 : MiniMaxは、以前のハイブリッド／スライディングウィンドウ方式がマルチホップ推論で課題を抱えていたことを公表し、M2をフルアテンションに切り替えました。それでもM2は200kコンテキスト、約100TPS、幅広いツールチェーン対応で強力なエージェント型コーディング性能を示し、期間限定で無料試用可能です。コミュニティでは、以前の線形型M2はシンプルだったが、KDAのような効率的なハイブリッドはマルチホップ劣化が小さい場合有望との声も。リンク: @omarsar0、vLLM M2 support、@aicodeking、@SonglinYang4。
  • 潜在ループと適応計算 : ByteDance/Princeton/Milaの「Looped LLMs」は、1.4B/2.6B LoopLMs（7.7兆トークン）が標準トランスフォーマー4B/8Bとほぼ同等のベンチマーク結果を示し、ループ型潜在推論が品質とデータ効率を時間と引き換えにできる可能性を示しています。MoEとのスケーリングも期待されています。リンク: @iScienceLuvr、project/paper。また、DeltaNet/RetNet/Mamba-v2の減衰とデルタルール変種、MLAの部分RoPE/NoPEのトレードオフについても議論が盛り上がりました（例: @nrehiew_、@Grad62304977）。

エージェント型コーディングとツール利用システム

  • OpenAIのAardvark (GPT-5) がプライベートベータ開始 : Aardvarkは「エージェント型セキュリティ研究者」として、コードの読み取り・分析、テスト作成・実行、パッチ提案を行います。初期ユーザーは、脆弱性発見と修正のスケーラブルな未来を垣間見たと評しています。リンク: @OpenAI、@gdb、@embeddedsec。
  • コーディングエージェントの急速展開と検証 : Cursorは高速で信頼性の高いクラウドエージェントを公開し、内部利用パターンを共有しました（launch、how they use it）。一方、CursorのComposer-1が時折中国語で「思考」することがあり、ベースモデルの透明性に関する疑問が出ています（@auchenberg、@simonw）。Cognitionは「Computer Use」をパブリックベータで公開し、Devinがデスクトップ／モバイルツールを操作し、画面録画やGUIアプリ構築が可能になりました（@cognition）。
  • ツール利用評価とオーケストレーション : HKUSTのToolathlon（Tool Decathlon）は32アプリ／600以上のツールでの実行ベース評価を導入し、現SOTA性能が低いこと（例: Claude Sonnet 4.5で38.6%成功率）やオープン／プロプライエタリ間のギャップを明らかにしました（@junxian_he）。新しい計画手法として、RLベースのスケジューリングによる並列ツール利用（Graph-based Agent Planning、paper）も登場。LangGraphはリデューサーをバイパスして直接状態置換するOverwriteを追加（@caspar_br）。LangChainはノーコードAgent Builderのラウンドテーブルを公開（@LangChainAI）。
  • リアルタイムコンテキストパイプライン : イベント駆動型「ストリーミングエージェント」が、Confluent + Weaviateの事例やConfluent + Qdrantの提携により、ライブデータ＋ベクトル検索を活用し、古いRAGスナップショットを超えたコンテキスト対応エージェントを実現しつつあります（Weaviate、Qdrant）。

トレーニング、評価、エンベディング

  • Hugging FaceのSmol Training Playbook（200ページ以上） : HFサイエンスチームによるフィールドガイドで、事前学習データ選定、アーキテクチャ選択、事後学習（SFT/RL）、インフラデバッグ（NCCL問題も含む）を網羅。論文では省かれがちなアブレーションや現場の課題に焦点を当て、FineWebやUltrascale Playbookを補完します。リンク: @LoubnaBenAllal1、@_lewtun、@eliebakouch。
  • 企業向けエンベディング評価 : Voyageの量子化対応学習済みエンベディングモデルvoyage-3-largeが新しいHF RTEBリーダーボードで33データセット全てで1位を獲得し、OpenAIやCohereを金融・法律・医療の検索タスクで上回りました。QATによりINT8やバイナリでも精度を維持し、ベクトルDBコスト削減と高速推論を実現（@_avichawla）。
  • オープンとクローズの差が縮小 : Epoch AIのECIによると、オープンウェイトがクローズSOTAに追いつくまでの平均遅れは約3.5ヶ月（約7 ECIポイント）で、「pro」と「mini」の差と同程度。想定よりも早いキャッチアップを示しています（@EpochAIResearch）。
  • 遅延インタラクション検索インフラ : LightOnのFast-Plaid 1.2.5は、ColPali/ColQwen/PyLateスタイルの検索で速度とGPUメモリ削減を実現（@raphaelsrty）。

マルチモーダル：音声、動画、画像編集

  • SSMベースの大規模音声 : Cartesiaの新TTS「Sonic-3」はState Space Model構造を採用し、笑いや驚きなどのプロソディ要素を含む低遅延ストリーミング音声を実現。42言語（インド系9言語含む）対応で、Artificial Analysisの評価アリーナに参加（@ArtificialAnlys）。
  • 物理認識型編集と世界シミュレーション : NVIDIAのChronoEdit-14B（コード・モデル・デモをオープンソース）は、動画推論ステージ＋軌跡トークンのインコンテキスト編集により、約8ステップの拡散で画像編集を実現（H100で約4秒／画像）。編集の「思考過程」可視化にも有用（paper/model/demo、author update）。
  • 動画生成アップデート : GoogleのVeo 3.1は画像から動画への変換性能が大幅に向上（Veo 3.1 FastがAAのI2Vアリーナで2位）しましたが、テキストから動画の品質はVeo 3と同等。価格は音声なしで$0.2/秒のまま（@ArtificialAnlys）。
