AI Twitterまとめ

AnthropicのClaude Opus 4.7発表：コーディングやエージェント性能強化、新トークナイザー、長文コンテキスト評価は賛否両論

  • Claude Opus 4.7 はAnthropicが発表した最新のOpusモデルで、長時間タスク、指示遵守、自己検証、コンピュータ利用ワークフローにおける性能向上が特徴です。第三者やエコシステムからの報告では、コーディングやエージェントベンチマークが大幅に改善され、価格はOpus 4.6と同じ（$5 / $25／100万トークン）、APIや製品全体に展開、新たにxhigh推論ティアが追加されました。
  • ベンチマークの差は特にソフトウェア工学分野で顕著。コミュニティのまとめでは SWE-bench Pro 64.3%、SWE-bench Verified 87.6%、TerminalBench 69.4%などが報告され、ValsではOpus 4.7が Vals Index 71.4%で複数評価項目で1位を獲得。Artificial Analysisも GDPval-AAで1753 Eloと発表しました。
  • 技術的変更点：複数の観測者が新トークナイザーを指摘し、単なる微調整ではなく新しい、または中間段階のベースモデルの可能性を示唆。画像入力解像度は約3.75MPに向上し、スクリーンショットを多用するコンピュータ利用エージェントに有利。Anthropicスタッフによれば、モデルはより多くの「thinking tokens」を使用し、購読者のレート制限も緩和されました。
  • 注意点と論争：一部ユーザーは長文コンテキストベンチマーク（特にMRCR／needle型検索）でスコア低下を指摘し、コーディング以外では実利用評価が混在。AnthropicのBoris ChernyはMRCRを優先度低下させ、Graphwalksなど実用的長文コンテキスト指標に注力していると説明。システムプロンプト強化や文字通りの指示遵守によるUX変化も議論されました。
  • 迅速な導入：Cursor、VS Code / @code、Replit Agent、Devin、Cline、Perplexity、Hermes Agentなどで数時間以内に対応が開始されました。

OpenAIのCodex拡張とGPT-Rosalind：エージェントワークスペース拡大とライフサイエンス特化モデル

  • Codexの機能範囲が大幅拡大。OpenAIはCodexをコーディング支援から、Macでのコンピュータ利用、アプリ内ブラウザ、画像生成・編集、90以上のプラグイン、マルチターミナル、SSHリモート開発環境、スレッド自動化、ファイルプレビュー、嗜好記憶などを備えた広範なコンピュータエージェントへと位置付け直しました。OpenAIはこれを「コードを書く前後や周辺の作業を支援する」と説明。
  • Anthropicとの製品戦略の違い：Anthropicはモデル能力の最前線を推進、OpenAIはエージェントワークスペース統合を重視。背景Mac制御やブラウザ／ドキュメント／画像ツールにより、Codexは純粋なコーディングを超えた用途に有用と評価されました。NVIDIAもこの方向性を支持。
  • GPT-RosalindはOpenAIのもう一つの発表で、生物学、創薬、トランスレーショナルメディシン向けの信頼アクセス型推論モデル。顧客にはAmgen、Moderna、Allen Institute、Thermo Fisherが含まれ、タンパク質・化学推論、ゲノミクス、生化学知識、科学ツール利用に最適化。
  • 解釈：Rosalindは単一のベンチマーク突破モデルというより、垂直特化型オーケストレーション／推論製品であり、最前線ラボがドメイン特化モデルラインとゲート付き展開構造へ移行している兆しです。

Qwen3.6-35B-A3Bとオープンモデル推進の継続

  • AlibabaがQwen3.6-35B-A3BをApache 2.0ライセンスで公開。総パラメータ35B、稼働3Bの疎MoEモデルで、ネイティブマルチモーダル対応、thinking／non-thinkingモードを搭載。小規模稼働パラメータで強力なエージェントコーディング性能を実現。
  • 性能評価はサイズクラスに対して注目に値。Qwen3.5-35B-A3Bや密Qwen3.5-27Bを上回るコーディングベンチマーク結果を報告。VLMベンチマークではClaude Sonnet 4.5に匹敵する性能を示し、特に空間認識タスクで高得点。
  • 展開面でも初日から強力：vLLM v0.19+でツール呼び出し、thinkingモード、MTP推測デコード、テキスト専用モード対応。OllamaやUnslothも即日ローカル対応を発表。
  • 広範な傾向：Qwenは小型・高性能・Apacheライセンスのモデルへの移行を象徴し、ローカルやインフラ効率的展開に適しています。

Cloudflareのエージェントインフラ推進：Git互換ストレージ、メール、推論統合

  • CloudflareがArtifactsを発表。エージェント向けGit互換バージョン管理ストレージで、各セッションに耐久性のあるファイルシステムを提供。開発者はこれをエージェントネイティブアプリの欠けていた基盤と評価。
  • Cloudflare Email Serviceがパブリックベータに入り、WorkersやRESTから直接送受信可能に。メールエージェント構築に有用。
  • 推論／プラットフォーム統合：Workers AIでホスト／プロキシモデルを単一バインディングで扱える統合プラットフォームを構築中。

エージェント、評価、オープンワールドベンチマークが実運用に接近

  • オープンワールド／実運用評価がテーマに。新プロジェクトCRUXはベンチマーク飽和を指摘し、長く複雑な実タスク評価を提案。初タスクではApple DeveloperアカウントとMac VMを使いiOSアプリを構築・公開に成功。
  • AlphaEvalは製品レベルのエージェント評価を推進、94タスクを7社から収集し、形式検証、UIテスト、ルーブリック、ドメインチェックなどを組み合わせ。
  • FrontierSWEはコーディングエージェント向けに平均11時間の超長期タスクに焦点を当て、最前線モデルでも失敗が多いと報告。
  • エージェント製品は記憶と共有状態を中心に厚みを増す：Nous/MiniMaxがMaxHermesを発表、Mirra Workspacesが共有マルチテナント環境とスキル同期を導入、NousがPortalでTool Gatewayを提供。

AI Redditまとめ

/r/LocalLlama + /r/localLLMまとめ

1. Qwen3.6-35B-A3Bモデル発表とベンチマーク

  • Qwen3.6-35B-A3Bは疎MoE構造で総パラメータ35B、稼働3B、マルチモーダル対応。コーディング・推論性能で前モデルや同サイズ競合を上回り、Apache 2.0ライセンスで公開。RefCOCOやODInW13など視覚言語ベンチマークでも高得点。
  • コメントではGemmaやGoogle大型モデルとの競争を予想する声もあり、今後の大型版リリースに期待が寄せられています。

2. ローカルAIとモデル利用

  • ローカルAIは検閲やデータ収集なしで利用でき、プライバシー重視ユーザーに魅力的。高性能GPU環境での大規模モデル運用や、APIコスト削減の利点が強調されました。
  • セットアップやメンテナンスの複雑さが課題ですが、クラウドモデルの性能低下に対しローカルモデルは改善傾向にあり、実用化が進むと予想されています。

3. Gemmaモデル改善と利用

  • Gemma 4 E4BとGemma 4 26Bを組み合わせたルーティングとタスク処理が、以前のQwen構成より効率的で正確と報告。
  • システムプロンプト改変による検閲解除や成人コンテンツ対応の議論もあり、モデルの自由度向上を求める声が見られます。

技術色の薄いAI Subredditまとめ

1. Claude Opus 4.7発表とベンチマーク

  • Opus 4.7は長時間タスク処理や自己検証能力を強化し、画像解像度も大幅向上。ただし長文コンテキスト検索性能は低下し、MRCRからGraphwalksへの移行が説明されています。
  • 価格はOpus 4.6と同じで、Amazon Bedrock、Google Vertex AI、Microsoft Foundryなどで利用可能。

2. AIモデルのロールプレイ・創作利用

  • RP用途でのモデル選択に苦労するユーザーが多く、各モデルの長所短所を組み合わせたいという要望が見られます。
  • DeepSeekは長期記憶性能が改善され、数百メッセージ後も詳細を保持するとの報告があります。

3. AIのコーディング・開発ワークフロー利用

  • Claude Codeの効率的利用法として、計画モードや小ステップ実装、自己バグ修正、セッション終了時の振り返りなどが推奨されています。
  • AIによる高速化で開発ペースが上がり、意思決定負荷や疲労感が増しているとの声もあり、従来のスキル価値低下を懸念する意見が見られます。
  • AMDエンジニアによるClaude Codeセッション分析では性能変化やコスト増が明らかになり、Anthropicが一部を認め対策を提示しました。
  • Boris Cherny氏はClaudeをチャットボットではなくマルチエージェントシステムとして活用し、複数インスタンス並行稼働や自動テスト・修正サブエージェントなどを駆使しています。
