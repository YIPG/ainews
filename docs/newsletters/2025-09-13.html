AI Twitter Recap

オンデバイス推論強化：MetaのMobileLLM-R1（サブ1B）がHFで公開

  • MobileLLM-R1（サブ1B、オープンウェイト）：MetaはHugging Face上でサブ1Bパラメータの推論モデル群を公開しました。小規模モデルとしては異例の高精度で、MATH精度はOlmo‑1.24B比で約5倍、SmolLM2‑1.7B比で約2倍、複数の推論ベンチマークでQwen3と同等または上回る性能を示しました（学習トークンは4.2TでQwen3の約11.7%）。発表、詳細も参照ください。Anycoder/Spaces経由でのコミュニティデモもすぐに登場しました（アプリ、別デモ）。

Qwen3‑Next‑80B（A3B）：ハイブリッドアテンション、256kコンテキスト、インフラ影響大

  • アーキテクチャと推論の複雑性：Alibabaの新しいオープンウェイトモデルQwen3‑Next‑80B‑A3Bは、Gated DeltaNet＋Gated Attentionのハイブリッド設計を採用し、高スパース化（アクティブパラメータ約3.8%、Qwen3‑235Bは9.4%）を実現。256kのコンテキストウィンドウを持ち、テキスト入出力専用です。SGLangでは6k行超、vLLMでは2.5k行超のコード変更が必要でした。Alibaba Cloudでの価格は推論付きで入力100万トークンあたり$0.5、出力$6、推論なしは出力$2とQwen3‑235Bより安価です（詳細）。
  • 性能とトレードオフ（コミュニティ評価）：長期的な「作業記憶」やマルチターンの一貫性が向上。文字レベルの基礎は強い一方、推論＋文字タスクは混在。弱点としてはエラー継承、指示追従の不足、長文での幻覚などが指摘されています（要約）。

エージェント、評価修正、失敗分析

  • SWE‑Bench修正：FAIR Codegenの@TacoCohenが、エージェントが未来のコミットを参照できる不具合を指摘し、SWE‑Benchが即修正。再実行では多くのモデルに大きな影響はなし。今後は修正版ベンチマークでの再公開と明記が推奨されています。
  • ライブ評価の難しさ：LiveMCP‑101はリアルタイムの複雑タスク評価フレームワーク。最先端モデルでもGPT‑5が「難易度高」タスクで39.02%、全体でも60%未満。失敗パターン7種を分類（要件無視、過信、ツール選択ミス、構文/意味/出力解析エラーなど）。
  • 推測よりキャリブレーション：OpenAIは、ベンチマークが自信満々の推測を報奨するため幻覚が残ると指摘。「わからない」を罰しない評価やリーダーボードの再調整を提案。AssistantBenchではGPT‑5がo3より高精度・低推測率を示しました。

ツール、インフラ、ライブラリ

  • VS CodeにモデルマーケットプレイスAPI：Language Model Chat Provider拡張APIが完成。BYOKプロバイダを拡張としてインストール可能に。Claude、GPT‑5/mini、Geminiなどの自動選択も搭載。
  • Transformers v5＋連続バッチ処理：Hugging Faceがv5の高速化やデフォルト改善を予告。連続バッチ処理も追加され、評価や学習ループが簡素化。
  • 推論システム：MetaのvLLM分離推論が内部スタックより低遅延・高スループットを実現。Paged Attentionの解説も共有されました。
  • AOTとリージョンコンパイル：ZeroGPUが地域別AOTコンパイルと事前コンパイルグラフの共有/読み込みを追加。
  • HFでのVision＆検索：MicrosoftのKosmos‑2.5がTransformersに追加され、OCR＋レイアウトデモを提供。MetaCLIP2多言語モデルとテキスト検索ノートブックも登場。

フロンティアアクセス、SDK、安全性協力

  • OpenAIプラットフォーム：GPT‑5とgpt‑5‑miniのレート制限を大幅引き上げ。Codex‑CLIに新ターゲット「gpt‑5‑high‑new」が追加。
  • Anthropic：英国AISIと米国CAISIがClaude Opus 4/4.1の脱獄事例を特定し、安全性を強化。Claude Code SDKも公開。
  • Qwen Code：v0.0.10/11でサブエージェント、Todo Writeツール、プロジェクト要約、編集安定性向上、IDE/シェル統合改善などを追加。

ビジョンモデルとリーダーボード
