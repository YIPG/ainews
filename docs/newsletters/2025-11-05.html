AI Twitter Recap

Kimi-K2 がオープン推論スタックに登場、Perplexity が兆パラメータ規模の MoE カーネルを解放

  • Kimi-K2 Reasoner → vLLM と SGLang : Kimi-K2 推論モデルが vLLM に統合され、メンテナが「まもなく」と予告。SGLang もローンチ時にサポート予定。Kimi-K2 の MoE 構成は総計1.2兆パラメータ／アクティブ約300億パラメータとされ、最近の大規模スパースモデルに類似。発表1、発表2、発表3。
  • Perplexity のカスタム MoE カーネル（AWS EFA） : Perplexity が大規模 MoE 用の初の研究論文とカーネルを公開。AWS EFA 上で兆パラメータ規模のデプロイ（例：Kimi K2）が可能と主張。vLLM は高速通信カーネル統合を示唆。発表、解説、vLLMの反応。
  • vLLM v1 のハイブリッドモデルサポート : IBM の vLLM チームがハイブリッドモデル（dense + sparse experts）を正式サポート。Qwen3-Next、Nemotron Nano 2、Granite 4.0 などが対象。PyTorch発表、NVIDIA DGX Spark ガイド、EUミートアップ配信。
  • Kimi-K2 ベンチマーク（未確認） : GPQA Diamond で77%（GPT‑4.5は71.4%）との主張。出典。広範な評価が出るまで未確認扱い。

エージェントシステム、MCP、コーディングスタックの本番化

  • Anthropic のコード実行 + MCP パターン : MCPサーバをコードAPIとして扱い、ツールを段階的に発見し、環境内でデータ処理を行うことで、ツール重視エージェントを安価かつ高速化する方法を解説。コンテキストを約150k→約2kトークンに削減（約98.7%減）。解説。
  • アプリ間の共有メモリ（Graphiti MCP） : ローカル Graphiti MCP サーバを Claude Desktop と Cursor に接続し、ツール間で知識グラフを共有するデモ。解説とリポジトリ。
  • VS Code に「エージェントセッション」ビュー追加 : Copilot や外部エージェントを統合的に管理する新ビュー。発表1、発表2、発表3。
  • リポジトリ規模のコーディング精度向上＝検索精度 : Cursor が大規模コードベースで grep よりもセマンティック検索が有効と報告。解説、ブログ。
  • 実務的エージェント評価 :
    • CodeClash が複数ラウンドのリポジトリ進化タスクでモデルを対戦評価。解説1、解説2。
    • LMArena が職業別リーダーボード「Arena Expert」を公開。発表、分析。
  • その他：OpenHands Cloud 無料化、openenv によるRL環境共有、Voiceflow KB メタデータルーティング、Dify が Qdrant を統合。

マルチモーダルと動画：被写体一貫性、リアルタイム生成、制御性

  • ByteDance BindWeave（被写体一貫動画） : クロスモーダル統合による被写体一貫型 I2V。解説、論文、モデル。
  • 単一 H100 でリアルタイム動画生成 : MotionStream が約29FPS／0.4秒レイテンシを達成。解説。
  • 生成後のカメラ編集 : Google Veo 3.1 が生成済み動画のカメラ角度・動きを変更可能に。ユーザーテスト。Qwen Image Edit LoRA も強力なカメラ制御を提供。デモ、解説。
