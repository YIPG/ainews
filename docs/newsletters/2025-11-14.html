OpenAIのGPT‑5.1ローンチ：適応型推論、開発者向けAPI、新UXパイロット

  • 適応型推論＋開発者重視：OpenAIは「適応型推論」を備えた5.1‑Instantをリリースしました。これは新しいポストトレーニング／RL作業を必要とした変更であり、@allisontam_によると、開発者の使いやすさ向上にも注力しています。@gdbは「APIに優れた新モデルと拡張されたプロンプトキャッシュ」を強調しました。OpenAIチームは長時間稼働／エージェント的タスク（ツール計画、永続化）への具体的なガイドやPrompt Optimizerを公開し、推論負荷（reasoning_effort）を切り替えてレイテンシ／コストと品質をトレードオフできる機能も追加しました（OpenAI DevRel、tip 2、tip 3）。また、日本、ニュージーランド、韓国、台湾でChatGPTの「グループチャット」を試験運用しています（発表）。
  • ベンチマークとUXの変化：独立トラッカーArtificial Analysisによると、GPT‑5.1はGPT‑5に比べ「小幅な」知能向上（指数で+2）で、主にTerminalBench（エージェント的コーディング／ターミナル操作）で+12ppの改善が要因です。非推論エンドポイントは変化なし、価格も据え置きでややトークン効率が改善されています（@ArtificialAnlys）。体験談では、GPT‑5.1は創作やコード生成に強く（5.1‑Codex推奨： @gdb、VS Codeでのライブテスト： @JamesMontemagno）、カスタム指示のスタイル制約を守るなどステアラビリティも小幅改善しています（@sama）。

Anthropic／Claude：構造化出力、Claude Codeの勢い、Opus 4.5噂の否定

  • 構造化出力（パブリックベータ）：Claude APIがJSONスキーマ／ツール準拠の応答を保証し、再試行やパースの回避が可能になりました。Sonnet 4.5とOpus 4.1から開始（Haiku 4.5も近日対応）。ドキュメントとAPI例は@alexalbert__、続報参照。長年のユーザーは、これが以前の「単一ツール＋スキーマ」レシピを統合するものだと指摘しています（@simonw）。
  • Claude Codeによるエージェント運用：ログやテストスクリプト、厳密なハーネスを与えることで、Claude Codeはエージェントワークフローの最適化に効果的であると開発者が報告しています（@omarsar0、デモ）。コミュニティでは、Opus 4.1が依然として総合品質で優れているとの声が多いです（@abacaj）。
  • 噂の制御：「Opus 4.5」という記述がClaude Code CLIのPRに見られましたが、これはオートコンプリートや日付のアーティファクトであり、密かなリリースではないとされています（@scaling01、1、2）。政策分野では、Anthropicが政治的テーマの公平な扱いに関する規範ベースのアプローチを再確認しました（@AmandaAskell）。

エージェント、プロトコル、ツールチェーン：Deep Agents、MCPの誕生日、安全な自律UX

  • Deep Agents＋コンテキストエンジニアリング：LangChainが「Deep Agents」パッケージ／CLIを公開し、コンテキストエンジニアリングのプレイブックを共有しました（概要、ポッドキャスト）。
  • プロトコルによる相互運用性：CopilotKitは、LangGraph／CrewAI／Agno間でエージェントスタックを標準化する3つの補完的なオープンプロトコル（AG‑UI、MCP、A2A）を紹介しました（@_avichawla、PDF）。
  • MCP1周年：AnthropicとGradioがMCPの1周年ハッカソンを開始、6,300人以上が登録、2週間、賞金$20k＋クレジット$3.5M（@Gradio、@huggingface）。
  • エージェントUXの信頼と制御：PerplexityのComet Assistantは、ログインや購入などのセンシティブな操作前に一時停止し、トレースを表示し、明示的な許可を要求するようになりました（@perplexity_ai）。
  • エージェント的コーディングプラットフォーム：ClineがNousのHermes 4 70B／405BをVS Code、JetBrains、CLIで直接サポートしました（@NousResearch、@cline）。

開発ツールとインフラ：リポジトリQA、コーディングエージェント、実験運用

  • リポジトリ対応コードQA：Googleの「CodeWiki」は自然言語でコードベースを検索でき、DSPyリポジトリで良好な結果を示しました（@getpy）。
  • Qwen Code v0.2.1：17日間で8回のリリースを行い、無料ウェブ検索（複数プロバイダ、OAuthユーザーは1日2,000回）、ファジーマッチングによる再試行／トークン使用削減、プレーンテキストツール応答、Zed IDE改善、スマートファイルフィルタリングと.gitignore対応、Unicode修正、性能最適化を実装しました（@Alibaba_Qwen）。
  • 実験運用：SkyPilotがWeights & Biasesとネイティブ統合し、マルチクラウド／K8s起動、障害回復、共同トラッキングを可能にしました（@skypilot_org）。W&BはLEETというターミナルUIをリリースしました（クリップ）。
  • IDE／ランタイム改善：VS Codeがチャットフロー内で失敗したコマンドのインラインターミナル出力を追加（@Tyriar）。ColabとVS Codeの統合がイベントで注目されました（@patloeber）。DojoがOpenEnv仕様をサポート（@chakra_ai）。CUDA Graphsによる高速化報告が続く一方、record_streamの誤用には注意が促されています（@maharshii、@vikhyatk）。
