AI Twitterまとめ

オープンモデル：Qwen3.6-27B、OpenAI Privacy Filter、Xiaomi MiMo-V2.5

  • Qwen3.6-27Bがローカル／オープンの本格的なコーディングモデルとして登場 : @Alibaba_Qwen が Qwen3.6-27B を公開しました。これは dense、Apache 2.0 ライセンスのモデルで、thinkingモード／non-thinkingモード、統合マルチモーダルチェックポイントを備えています。Alibabaによれば、より大規模な Qwen3.5-397B-A17B を主要なコーディング評価で上回り、SWE-bench Verified 77.2 vs 76.2、SWE-bench Pro 53.5 vs 50.9、Terminal-Bench 2.0 59.3 vs 52.5、SkillsBench 48.2 vs 30.0 を達成しています。また、画像や動画に対するネイティブな視覚言語推論もサポートしています。エコシステムは即座に対応し、vLLMが初日からサポート、Unslothが18GB RAMローカルGGUFを公開、ggmlがllama.cpp対応を追加、Ollamaがパッケージリリースを提供しました。@KyleHessling1 や @simonw からの初期ユーザ報告では、ローカルフロントエンド／デザインや画像タスクで非常に高評価でした。

  • OpenAIが実用的なプライバシーモデルを静かにオープンソース化 : 複数の観測者が、OpenAIの新しい Privacy Filter を指摘しました。これは軽量な Apache 2.0 オープンモデルで、PII検出とマスキングを行います。 @altryne、@eliebakouch、@mervenoyann によれば、1.5B総パラメータ／50MアクティブMoE のトークン分類モデルで、128kコンテキストウィンドウを持ち、大規模コーパスやログに対する低コストのリダクションを目的としています。これは単なる小型オープンモデルではなく、企業やエージェントパイプラインでオンデバイスや低コスト前処理が重要となる具体的なインフラ課題を狙ったリリースです。

  • Xiaomiがエージェント型オープンモデルを強化 : @XiaomiMiMo が MiMo-V2.5-Pro と MiMo-V2.5 を発表しました。Xiaomiは V2.5-Pro をソフトウェアエンジニアリングと長期エージェントにおける大きな飛躍と位置付け、SWE-bench Pro 57.2、Claw-Eval 63.8、τ3-Bench 72.9 を達成し、1,000以上の自律ツール呼び出しを可能とするとしています。非Proモデルはネイティブオムニモダリティと100万トークンコンテキストウィンドウを追加。ArenaはすぐにMiMo-V2.5をText/Vision/Code評価に掲載、Hermes/Nous統合も @Teknium により実施されました。

Google Cloud Next: TPU v8、Gemini Enterprise Agent Platform、Workspace Intelligence

  • Googleのインフラ発表は見た目だけでなく実質的 : @Google と @sundarpichai は、第8世代TPU を発表しました。設計は分割され、TPU 8t はトレーニング用、TPU 8i は推論用です。Googleによれば、8t はIronwood比でポッドあたり約3倍の計算能力を提供し、8i はポッドあたり1,152 TPUを接続して低レイテンシ推論と高スループットのマルチエージェントワークロードを実現します。 @scaling01 は、TPU8tで単一クラスターに最大100万TPUをスケールできるという追加主張を指摘しました。ハードウェア性能と同様に製品化の方向性も重要で、Googleはチップ、モデル、エージェントツール、エンタープライズ制御プレーンを垂直統合した提供に明確に向かっています。

  • エンタープライズエージェントがGoogleの主要製品領域に : @GoogleDeepMind と @Google は Gemini Enterprise Agent Platform を発表しました。これはVertex AIの進化形として、大規模なエージェント構築・管理・最適化のためのプラットフォームです。Agent Studio、Model Garden経由で200以上のモデルへのアクセス、Googleの現行スタック（Gemini 3.1 Pro、Gemini 3.1 Flash Image、Lyria 3、Gemma 4）のサポートを含みます。関連リリースとして、Workspace Intelligence GA、Gemini Enterprise inbox/canvas/reusable skills、Agentic Data Cloud、Wiz統合によるセキュリティエージェント、Gemini Embedding 2 GA などが発表されました。

エージェント、ハーネス、トレース、チームワークフロー

  • 「エージェントハーネス」抽象化が各ベンダーで強化 : OpenAIはChatGPTのワークスペースエージェントを導入しました。これはチームで共有できるCodex搭載エージェントで、ドキュメント、メール、チャット、コード、外部システムに対応し、Slackベースのワークフローやスケジュール／バックグラウンドタスクも可能です。GoogleもGemini Enterprise Agent Platformで同様の動きを見せ、CursorはSlack呼び出しによるタスク開始とストリーミング更新を追加しました。パターンは収束しつつあり、クラウドホスト型エージェント、チーム共有コンテキスト、承認、長時間実行が主流になっています。

  • ハーネス／モデル独立性における開発者の利便性向上 : VS Code/Copilotはキー／モデル持ち込み機能を全プランで提供し、ビジネス／エンタープライズにも対応しました。Anthropic、Gemini、OpenAI、OpenRouter、Azure、Ollama、ローカルバックエンドなどが利用可能です。 @omarsar0 は、多くのモデルが依然として自社エージェントハーネスに過適合していると指摘しました。Cognitionの Russell Kaplan は、企業顧客はモデルの柔軟性とSDLC全体に対応するインフラを求めており、特定ラボへの依存を望まないと述べています。

  • トレース／評価／自己改善がエージェントデータの中核に : LangChain関連の議論が強く見られました。 @Vtrivedy10 は、トレースがエージェントのエラーや非効率を捉えると述べ、計算資源をトレース理解に向けてより良い評価、スキル、環境を生成すべきと主張しました。詳細な続報では、トレースマイニング、スキル、コンテキストエンジニアリング、サブエージェント、オンライン評価を含む具体的なループを提案しています。 @ClementDelangue は、オープンエージェント訓練に必要なデータ基盤としてオープントレースを推奨し、@gneubig はADP / Agent Data Protocol標準化を推進しました。LangChainは @hwchase17 により、より強力なテスト／評価製品方向を示唆しました。
