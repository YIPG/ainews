NVIDIAのNemotron 3 Ultraと3.5 ASRリリース

  • Nemotron 3 Ultra は本日の中で最も明確な技術的リリースでした。完全にオープンな 550B MoE モデルで、55Bアクティブパラメータ、1Mコンテキスト を備え、長時間稼働するエージェントワークロードに特化しています。NVIDIAによれば、エージェントタスクにおいて最大5倍高速、30%低コストであり、ウェイト、合成データ、報酬チェックポイント、量子化バリアント、トレーニングレシピをOpenMDW 1.1ライセンスで公開しています（NVIDIA launch、NVIDIAAI open artifacts、Pavlo Molchanov thread）。アーキテクチャはハイブリッドMamba/attention、LatentMoE、native MTPを組み合わせ、NVFP4で20Tトークンの事前学習を実施しています。これは低精度事前学習を新たなスケール領域に押し広げた点で注目されています（tech notes、scaling discussion）。

  • ベンチマークと提供体制はオープンリリースとしては異例の強さでした。 @ArtificialAnlys は、NVIDIA推奨のNVFP4推論ウェイトでIntelligence Index 47.7（BF16では48.2）を測定し、米国のオープンウェイトモデルとして最も高い性能を記録しましたが、Kimi K2.6には及びません。さらに、BlackBox経由で400+出力トークン/秒を報告し、Nemotron 3 Ultraがタスク遅延と性能のParetoフロンティア上に位置することを示しました（latency analysis、BlackBox throughput）。モデルは初日からvLLM、Modal、Together、Fireworks、Ollama cloud、Baseten、CoreWeave/W&B、Cline、Prime Intellect、Nous Portalなどで提供開始されました。

  • Nemotron 3.5 ASRはより静かですが実用的なリリースです。単一の0.6Bチェックポイント、40言語ロケール、100ms未満の遅延を持つオープンストリーミングASRモデルで、cache-aware FastConformer / RNN-T設計を採用し、音声エージェントやストリーミング音声ワークロードに最適化されています（Piotr Zelasko、Together、fal availability）。

Anthropicの再帰的自己改善（RSI）フレーミングと内部AIコーディング指標

  • Anthropicは最も議論を呼んだ政策・研究ノートを公開し、現行システムが再帰的自己改善（RSI）の初期兆候を示していると主張しました。研究方向の完全な自律性には至っていませんが、AIがAI開発を加速している明確な証拠があるとしています（Anthropic post）。具体的な運用データとして、Anthropicではマージされたコードの80%以上がClaudeによって書かれ、エンジニアは前年より8倍多くのコードを四半期ごとに出荷し、内部のオープンエンドなエンジニアリングタスク成功率は6か月で約26%から76%に上昇しました（code metric、Alex Albert summary）。

  • 最も注目された実証データは、Anthropicが繰り返し行っている「小規模モデルのトレーニングスクリプトを高速化する」テストで、Claude Opus 4は約3倍の高速化を達成し、Mythos Previewは約52倍を記録しました（Anthropic benchmark claim、correction on dates）。また、Mythosは研究者が誤った方向に進んだ際に「次に何をすべきか」という提案で人間より64%優れていたと報告しています（research-next-step result）。Anthropicの広範な見解としては、_問題選択_の自動化はまだ解決されていませんが、実装や反復の大部分の自動化はすでに進行中です。

  • ガバナンスの観点も生産性の主張と同じくらい重要でした。Anthropicは「フロンティアAI開発を遅らせる、または一時停止する選択肢が世界にあることは良いことだ」と明言し、RSI的な動態が続く場合、検証や協調メカニズムがますます緊急になると述べています（Anthropic governance statement、discussion、commentary）。これは、Anthropicが最近バイオ・化学リスクに関するResponsible Scaling Policyの一部基準を弱めたとの批判の中で発表されました（@CRSegerie）。別途、Altman、Amodei、Hassabis、Bakerを含む連合が、AIが生物学的知識障壁を低下させているとして、米国でのDNA合成の義務的スクリーニングと記録保持を支持しました（letter summary）。

CloudflareによるVoidZero買収とフルスタックエージェントツールチェーン強化

  • 最大の開発者プラットフォーム動向は、CloudflareがVoidZero（Vite、Vitest、Rolldown、Oxc、Vite+の開発チーム）を迎え入れたことでした。CloudflareとVoidZeroは、Viteはオープンソース、MIT、ベンダーニュートラルであり続けることを強調し、Cloudflareは独立したViteエコシステム開発のために100万ドルの基金を設立すると発表しました（Cloudflare、Vite statement、Evan You）。

  • 開発者の戦略的見解として、この動きはCloudflareにフロントエンド/ビルドツール、ランタイム、ストレージ、推論、デプロイ基盤、セキュリティを一元的に提供する、エージェントに優しいアプリケーションスタックの制御を強化するものとされています。 @wesbos は、Cloudflareが「LLMに渡せばサイトを作れる整ったパッケージ」を組み立てていると表現し、これはCloudflare自身のエージェント、MCP、サンドボックス、AI検索、決済、監視を統合したプラットフォーム推進と一致しています（Cloudflare agents docs overview）。

エージェント、ハーネス、メモリ、評価基盤

  • 複数の投稿が、生のモデルリリースを超えた「エージェントシステム」層の成熟を指摘しました。繰り返し出てきたテーマは、ボトルネックが単なるプロンプトではなくハーネス/オーケストレーターになっているという点です。人気のクリップではClaude Codeのワークフローを「もうClaudeにプロンプトしない、ループを書く」と要約し、 @omarsar0 は動的ワークフローを逆解析して、分岐研究、検証、トリアージ、データ合成、評価生成のための独自オーケストレーターを構築したと述べています。共通の考え方は、一回限りのプロンプトではなく、高次の制御ループが実際の作業単位になっているということです。

  • これらのループ周辺のツールも進化しました。LangSmith Sandboxes はDockerfileスナップショット、インタラクティブコンソール、TCPトンネリング、標準Linuxツールを備えてGAに到達しました。Hugging Faceは、Hub上でカスタムカーネルを配布するKernels経路（announcement）と、エージェントトレースを第一級アーティファクトとして保存する機能強化を推進しました（ @ClementDelangue）。 @julien_c は、オープンモデルをコーディングエージェントとして動作させ、ローカルモデルがユーザーをシミュレートすることで2,000+の合成コーディングエージェントセッショントレースを生成する最小ハーネスSynthTracesを公開しました。

  • 評価も現実世界のエージェント作業にシフトしました。ArenaはAgent Arena / Agent Modeを開始し、ウェブ検索、ファイルシステム、bash、画像生成などのツールを使った数百万のライブセッションからエージェント性能を測定しています。現在のランキングではGPT-5.5が1位、次いでClaude Opus 4.7、GLM-5.1、Gemini 3.1 Pro、Kimi-K2.6となっています（launch、methodology）。企業向けでは、CognitionがDevinに対してAI生産性保証を導入し、製品がエンジニアリング価値を生まない場合に最大1,000万ドルを保証する制度を発表しました（guarantee、technical writeup）。

メモリ、マルチモーダル、モデル/ベンチマーク更新

  • OpenAIはより高性能なChatGPTメモリシステムを米国のPlusおよびProユーザーに提供開始しました。メモリ要約、より多くの制御機能、2倍のメモリ容量を備えています（OpenAI、controls、Christina Kim explanation）。開発者向けの関連更新として、ResponsesおよびCompletions APIにモデレーションスコアが追加され、新しいCodex iOSアプリプラグインのデモも共有されました（OpenAIDevs demo）。

  • その他のモデル/データリリースとして、Gemma 4 12Bはローカルコーディングモデルの代替として注目され続けており、Unslothが2-bit GGUF（4.66 GB）を公開しました。 @_philschmidは、Gemma 4がテキスト/画像/音声を別のエンコーダなしで処理するアーキテクチャ解説を紹介しました。マルチモーダル研究では、@skalskip92がCVPRでMolmo2を強力なオープンVLM候補として指摘し、動画の指差し、追跡、カウント、複数画像推論をサポートしています。ドキュメント理解では、LlamaIndexがParseBenchを導入し、2,000+の人間検証ページと167K+のテストルールを含むオープンベンチマークを提供しています（benchmark announcement）。
