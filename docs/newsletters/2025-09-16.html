エージェント型コーディングとIDE：GPT‑5 Codexの展開、IDEコンテキスト、MCPの普及

  • GPT‑5 Codex、大きな適用範囲と賛否ある開発体験：開発者からは、エージェント的な能力やフロントエンド生成デモに感銘を受けた一方で、ハーネスの癖や長時間ループに不満という声もあります。ポジティブな例としては、CodexエージェントでReactアプリやアニメーション動画をエンドツーエンドで構築する事例（@gdb、@OpenAIDevs）。批判的な声としては、トークンの肥大化やループ、制御の不明瞭さ（@Teknium1、@finbarrtimbers）。OpenAIのインフラパートナーは需要増によるスループット低下を指摘（@thsottiaux）。分析としては、Codexは「重要な部分に労力を割く」設計で、難しい問題に多くのトークンを使い、レイテンシを犠牲に品質を取っているとのこと（@TheTuringPost）。
  • IDEスタックのアップグレード：VS Code InsidersがGPT‑5とClaude Sonnet 4向けに20万トークンのコンテキストを試験中（@pierceboggan）。GitHub MCP RegistryがVS Codeに統合され、ワンクリックでサーバー発見が可能に（@code）。Cursor 1.6はカスタムコマンド、高速なエージェントターミナル、MCPリソース、/summarizeを追加（@cursor_ai）。GitHub Copilotはタスクごとにモデルを自動選択（パブリックプレビュー）（@amandaksilver）。Perplexity ProはGmail/Calendar/Notion/GitHubのネイティブコネクタを提供、EnterpriseではLinear/Outlookも追加（@perplexity_ai、@AravSrinivas）。

推論・学習インフラ：vLLMのaarch64/GB200対応、ROCmアップデート、TRLのCP、Mac MLX高速化

  • vLLM 0.10.2が公式aarch64対応（NVIDIA GB200で動作）し、マルチプラットフォームDockerイメージを提供。今後さらなる性能改善予定（@vllm_project）。KV/QKキャッシュのボトルネックとPagedAttentionの仕組みを解説するスレッドも拡散中（@athleticKoder）。
  • ROCmの大規模アップグレード：AMDが最新のアテンション変種、スパースMoE、分散推論、RL/推論サポートを含む広範なスタック更新を発表。ノートPC/デスクトップにも対応（@realSharonZhou）。
  • 長コンテキスト学習向けのコンテキスト並列化：TRLがCPを追加し、GPU間・ノード間でシーケンスを分割可能に。Accelerateと統合（@SergioPaniego）。Hugging Face TransformersはMoEをネイティブカーネルにリファクタリングし大幅な性能向上（@art_zucker）。
  • RLとロボティクスのデータ基盤：Unsloth + vLLMの重み共有でマルチモーダルRLのVRAM使用量を50%以上削減、長コンテキストや数学/論理VLMの報酬設計を可能に（@danielhanchen）。LeRobotDataset v3はチャンク化エピソード、効率的な動画ストリーミング、OXE規模学習向けのparquetメタデータを導入（@LeRobotHF）。
  • Mac MLXの速度：Qwen3‑Next‑80B 4ビットがM4 Max 64GBで約66トークン/秒、使用メモリ約41GB（@rwojo）。LM StudioがMLX対応Qwen3‑Nextを追加し、バッチ生成デモで高いマルチストリーム性能を示す（@lmstudio、@awnihannun）。

新モデル・エージェント・空間知能

  • HunyuanImage 2.1（Tencent）：17B DiTテキスト→画像モデル、2048×2048ネイティブ、バイリンガル。Artificial Analysis arenaでHiDream‑I1‑DevやQwen‑Imageを上回る。「オープンウェイト」だがTencent Community LicenseによりEU/UK/KRでの利用、MAU1億超の製品、非Hunyuanモデルの学習用出力利用を禁止。HFデモやFALで利用可能（1,000画像あたり100ドル）（@ArtificialAnlys）。
  • Reka Speech：既存モデル比8〜35倍のスループットを持つ効率的なASR/翻訳モデル。Common Voice 16.1や社内STテストでWhisper‑Large v3を上回る精度。技術的には、プレフィル時にQ/KをCPUにオフロードし、生成後にアテンションを再計算してタイムスタンプを整合（@RekaAILabs、@artetxem、@_yuqiwang）。
  • Tongyi DeepResearch（Alibaba）：OpenAIのDeep Researchに匹敵するとされるオープンソースWebエージェント。パラメータ数30B（MoEで3B有効化）。Humanity’s Last Examで32.9、BrowseCompで45.3、xbench‑DeepSearchで75.0を記録（@Ali_TongyiLab）。
  • World Labs「Marble」3Dワールド：画像やテキストから永続的かつ大規模な3Dワールドを生成。公開ギャラリーあり。空間的な一貫性とスケールで大きな進歩を示す（@drfeifei、@theworldlabs、@jcjohnss）。
