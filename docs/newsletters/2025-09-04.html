オンデバイス埋め込みと検索スタックの最新情報

  • Google の EmbeddingGemma（308M）が一般公開 : Google/DeepMind が、オンデバイス RAG やセマンティック検索向けに設計された小型多言語埋め込みモデルを発表しました。特徴：308M パラメータ、500M 未満のオープンモデルで MTEB トップランク、100 以上の言語で学習、量子化により 200MB 未満の RAM で動作、Matryoshka 埋め込み（出力次元 768→128）対応、2k コンテキスト、一部設定で EdgeTPU レイテンシ 15ms 未満。Hugging Face Sentence Transformers、Ollama、MLX、llama.cpp、LlamaIndex、LangChain、Weaviate、Cloudflare Workers などで即時サポート。詳細と導入方法: @GoogleDeepMind 他。
  • Jina コード埋め込み（0.5B/1.5B）+ GGUF : コード特化の新しい埋め込みモデル（1〜4bit GGUF 量子化）が、15 以上の言語と 5 タスク（nl2code、code2code、code2nl、code2completions、QA）で SOTA 検索性能を主張。Qwen2.5-Coder の事前学習（5.5T トークン、92+ 言語）を基盤に、限られたペアでコントラスト学習。リンク: @JinaAI_ 他。
  • 蒸留なしの大規模検索学習 : LightOn の PyLate は、GradCache + 分散インフラを用いて数十億パッセージで直接コントラスト学習を行い、教師モデルなしで BEIR/BRIGHT の汎化性能向上を報告。概要: @LightOnIO。

ビジョン・言語データとマルチモーダルモデル

  • FineVision データセット（Hugging Face） : VLM 学習用の大規模オープンデータセットを公開。1,730 万枚の画像、2,430 万サンプル、8,890 万ターン、95 億トークン、200 以上の厳選ソース。10 ベンチマークで平均 20% 以上の性能向上を報告し、GUI ナビゲーション、ポインティング、カウント機能を追加。発表: @lusxvr 他。
  • MiniCPM‑V 4.5（8B）動画/画像 VLM : OpenCompass 平均 77.0（8 ベンチマーク）を達成し、GPT‑4o‑latest や Gemini‑2.0 Pro を上回ると主張。統一 3D‑Resampler と動画トークン圧縮（96×）を導入。デモ: @_akhaliq 他。
  • その他注目: Microsoft の VibeVoice TTS（7.5Hz の連続音声トークナイザー）、Stanford の Mixture‑of‑Contexts（1 パスで分単位の動画生成）。

オプティマイザ、内部メトリクス、学習レシピ

  • 堅牢なオプティマイザ比較（Marin プロジェクト） : Muon、Soap、Mars、Sophia、ScheduleFree、AdEMAMix、Prodigy などをモデル規模（0.1B–1.2B）、バッチサイズ、スケジューラ別に比較。AdamW に対する速度向上は大規模では減少傾向（1.2B で約 10%）。
  • 大規模学習における「内部メトリクス」（Kimi/K2） : 損失、勾配ノルム、出力 RMS、最大ロジットなどの内部信号を監視し、不安定化を防ぐ重要性を強調。MuonClip は最大ロジット制御で学習崩壊を回避。
  • Qwen3‑32B の創作文章ファインチューニング : WritingBench スコアを 78.97 → 82.08 に向上。カリキュラム SFT + DPO（RAFT）で CN‑EN コードスイッチ、繰り返し、推論を改善。推奨温度は約 0.6。
  • インフラ補足: slime RL フレームワークが Qwen3‑30B‑A3B の更新時間を 60s → 7s に短縮。

エージェントシステム、ランタイム、ツール
