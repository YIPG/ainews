AI Twitterまとめ

最前線のコーディングエージェントとモデル順位（Claude 4.5, Grok Code Fast, GoogleのJules, Qwenの命名, Arenaリーダーボード）

  • Claude Sonnet 4.5（実機レビュー）：約30時間Claude Codeを試した@finbarrtimbers氏は、Sonnet 4.5は「基本的にOpus 4.1と同じ」コーディング性能で、UXは洗練されているがGPT-5 Codexほどではないと評価。またChatGPT TeamのコストパフォーマンスがClaude Maxより高いと指摘。AnthropicはSonnet 4.5のサイバーセキュリティ性能（Opus 4.1と同等または一部で優位）と防御的能力への注力を強調しています（@AnthropicAI、続報）。
  • xAI Grok Code Fast：@gauravisnotme氏は、Claude 4.5やGPT-5 Codexより低コストで「高難度編集成功率が高い」と主張。独立検証が望まれるが、多くのユーザーがコーディングエージェントを次トークン予測精度より編集信頼性で評価し始めています。
  • GoogleのJulesコーディングエージェントがプログラム可能に：1週間の発表を経て、Julesを「プログラム可能なチームメンバー」としてツールやCI/CD統合が可能な公開APIを提供（まとめ@julesagent、API発表、ドキュメント）。@GoogleLabsや@googledevsからも追加情報。
  • Qwenの命名整理：Qwenモデルファミリー（LLM, Coder, VL, Omni, Image）、instructとthinkingのバリアント、APIティア（Max/Plus/Flash）、日付付きマイナー更新、「Qwen3-Next」の存在理由を整理した有用な分類（@JustinLin610）。
  • ライブランキング：Vision/LM Arenaではトップ層が非常に僅差で、Sonnet 4.5（標準と32k Thinking）、Claude Opus 4.1、Gemini 2.5 Proが同率1位。OpenAIモデル（4o-latest, 4.5 preview, 5 high, o3）は1ポイント差以内（@arena、続報）。OpenRouterはGrok 4 Fastがドイツ語プロンプトで優位と報告（@OpenRouterAI）。

動画生成の急伸：Sora 2 Proの勢い、評価、モデル群の拡大

  • Sora 2 Proの採用と性能シグナル：Sora 2がApp Storeで1位に。チームは招待を迅速に展開（@billpeeb）。高品質な15秒クリップが配信開始（@apples_jimmy）。初期テストではSora 2がGPQA形式の質問に約55%正答（小規模セット）、GPT-5は72%。動画生成前にLLMによる「プロンプト書き換え」層がある可能性（@EpochAIResearch、仮説、背景）。アプリはウォーターマーク除去ワークフローなど新たなクリエイターエコシステムも生み出しています（@angrypenguinPNG）。
  • エコシステムとベンチマーク：Video ArenaがLumaのRay 3とRay HDR 3を追加し、コミュニティ投票による評価を実施（@arena）。Kling 2.5は編集時のフレーム一致精度が高いと報告（@heyglif）。Veo-3類似のマルチモーダル動画＋音声生成「Ovi」が登場：5秒動画を24FPS、最大720×720、テキストまたはテキスト＋画像条件で生成（@_akhaliq）。

検索、VLM、認識モデル（ModernVBERT, Jina v3, RF-DETR, π0.5ロボティクス）

  • ModernVBERT / ColModernVBERT（MIT）：画像・テキストや文書検索用の小型双方向ModernBERTエンコーダで、ViDoRe上でColPaliと同等性能を約10倍少ないパラメータ（約2.5億）で達成。ColModernVBERTは遅延インタラクション型デュアルエンコーダでnDCG@5が+10.6、再ランクではなくサブ線形検索器として位置付け、10億文書のkNNを可能に（@pteiletche、@mervenoyann、HFモデル、著者スレッド、フレームワーク注記）。
  • リスト単位再ランク（Jina v3, 0.6B）：クエリと候補文書全てを一度に連結し、特殊トークン埋め込みを抽出してSOTAを達成（@JinaAI_、入力形式、リンク）。「ラストインタラクション」と銘打つが実際は早期の全コンテキストインタラクションで強い結果（@lateinteraction）。
  • 検出とセグメンテーション：RoboflowのRF-DETRセグメンテーションはCOCOでYOLO11-Lの3倍高速かつ高精度、T4上でTensorRT 10.4の低レイテンシ、DINOv3バックボーンで高性能（例：1エポックでの亀裂検出）（@skalskip92、レイテンシ、ノートブック）。
  • オープンソースロボティクスベースライン：Physical Intelligence π0とπ0.5がHugging Faceに公開され、PyTorch/LeRobotに完全移植。クロスエンボディメント・マルチ環境のVision-Language-Action訓練でオープンワールド汎化を目指す（@ClementDelangue）。
