Codexはこれだけで十分？

6月に取り上げたClaude Codeの静かな台頭と同様に、本日は通常ならトップニュースにはならないかもしれませんが、GPT‑5やOpenAIのCodex（Claude Codeへの回答であり、より幅広い機能を持つ）に対する1か月分の盛り上がりと好意的な評価の蓄積が注目に値します。さらに本日はOpenAIからの新発表もあり、これが勢いを後押ししています。詳細はこちらの記事で確認できます。Codexを多用していた方は、Discordセクションで指摘されている落とし穴にもご注意ください。

OpenAIのGPT‑5‑Codexとエージェント型コーディング競争

  • OpenAIがGPT‑5‑Codex（エージェント型コーディング）をリリース：OpenAIは、Codex CLI、IDE拡張、Web、GitHubコードレビュー、ChatGPT iOSなどで動作する、長時間稼働・ツール活用型のソフトウェアエンジニアリングに最適化されたGPT‑5のバリアントを発表しました。特徴は、動的な「タスク適応型」思考（簡単なタスクで15倍高速、難しいタスクで2倍慎重）、複雑なタスクでの7時間超の自律実行、命令遵守とコード品質の向上、SWE‑benchスタイルの性能改善などです。未公開の大規模「リファクタ」ベンチマークで51%の精度を達成したことや、SWE‑benchの比較改善にも言及しています。発表や議論は@OpenAI、@gdb、@samaなどで確認できます。初期の使用感は「より制御しやすく持続性がある」という声から、トークン消費や長いループに対する不満までさまざまです。さらに、XcodeサインインなどのOS統合も予告されています。
  • 評価とコーディング深度：OpenAIはSWE‑benchの改善や新しい内部評価「大規模リファクタPR」を発表しましたが、コミュニティからは公開版を求める声が上がっています。推論時の効率と品質には可変計算量とルーティングが重要であるとの認識が広がっています。

Qwen3‑Next 80B（A3B MoE）、長文コンテキスト、中国の効率化推進

  • Qwen3‑Next‑80B（3Bアクティブ）がTogether + NVIDIA NIMに登場：AlibabaのハイブリッドMoEモデルは、262kトークンのネイティブ長文コンテキスト（1M+まで拡張可能）、リポジトリ規模のコード解析、効率的な推論を狙っています。Together AIは“Instruct”と“Thinking”エンドポイントを提供し、NVIDIAはCUDA最適化されたアテンションでNIM対応を追加しました。Alibabaは「3Bアクティブパラメータのみ」での高性能を報告し、Gemini 2.5 Flash Thinkingとの比較結果も公開しています。Appleハードウェア上での高TPSも注目されています。
  • アーキテクチャ動向：ハイブリッドSSM + MoE：過去2週間で新しいMLX‑LMアーキテクチャの7件中6件がMoEであり、その半数がSSM/アテンションのハイブリッドです。中国と米国の学習環境の違いから、限られたFLOPSがインフラとモデルの共同設計、トークン効率、線形アテンション、推論時スケーリングへの注力を促しています。適切なレシピがあれば小規模モデルでも高性能化が可能という意見が広がっています。

エージェント向けツール：MCPの普及、Claude Code SDK、ワークフローの「vibe coding」

  • MCPの統合：Model Context Protocolは、M×Nのツール統合をM+Nに変える価値提案で注目を集めています。LangChainはニュースキュレーションやParserGPT、人間参加型のDeep Agentsなどのリアクティブエージェント例を公開しました。
  • Claude Code SDKがエージェント開発を効率化：Anthropicはコード参照、カスタムツール、フック対応を追加し、カスタムエージェントの開発を迅速化しました。ReplitのAgent 3（ノーコードワークフロー）やPoke（iMessageエージェントによる一時的サブエージェントのオーケストレーション）も登場しています。

推論・エージェント向けRL：オンラインRLの実運用、単一エージェントRL、学習手法の進化

  • オンラインRLの実運用：Cursorの展開は、継続的学習サイクルを数か月から数週間、さらには数時間に短縮する事例として注目されています。
  • 単一エージェントRLで複雑な構造に匹敵：長さ正規化報酬と戦略的ツール制限を組み合わせたシンプルなRL手法で、複数エージェント構成に匹敵する性能を達成する研究が報告されています。
  • HRLと分散RL：MetaのScalable Option LearningはGPU並列バッチ更新で25倍の学習速度を実現。GensynのSAPOは異種ノード群でロールアウトを共有し、累積報酬を最大94%向上させました。
  • 長期実行：小さな精度向上が長い推論チェーンで指数的に効果を発揮するとの分析が複数発表されています。

マルチモーダルとコンピュータ操作モデル

  • Holo1.5（オープンウェイト）：UIローカライズやQAでSOTAを達成する新しいVLM（3B, 7B Apache-2.0, 72B）を公開。オープンウェイト、クックブック、デモも提供されています。
  • Tencent SRPO：拡散モデルの全ノイズ除去過程で美的品質を最適化し、人間評価でリアリズム/美観を3倍以上向上。
  • MobileLLM‑R1（Meta）：0.14B〜0.95Bの小型推論モデルを発表し、140Mモデルはブラウザ内で完全動作。
  • 新データセット/ベンチマーク：SpatialVID（3Dアノテーション付き7,000時間超の空間ビデオ）、IntrEx（教育対話の興味深さラベル）を公開。

システムとインフラ（スループット、ルーティング、デプロイ）

  • スループットとプラットフォーム対応：FireworksはB200上でGPT‑OSS‑120Bを540 tokens/sで動作させ、主要ASICを上回ったと報告。vLLM 0.10.2はaarch64対応を追加し、Ray 2.49はKVキャッシュヒット率維持のためのルーティングを導入。
  • バッチ処理とクラスタ：Togetherはバッチ推論APIを刷新し、レート制限を3,000倍に拡大。Prime IntellectはGPUクラスタの予約インスタンスを提供開始。
  • カーネルとApple側の高速化：Standard Kernelは特定演算でcuBLAS/FlashAttention3を上回る最小CUDA+PTXカーネルを発表。MLXはM3 Ultraでの高TPSバッチ処理や評価時間短縮を実現。
  • Qwenのデプロイ活用：NVIDIAがQwen3‑Next NIMを追加し、BasetenやTogetherが本番利用に統合。
