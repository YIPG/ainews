AI Twitterまとめ

Agentic RLインフラとトレーニング後のトリリオンパラメータ規模

  • Prime Intellectのprime-rl v0.6.0は、今回の中で最も技術的に重要なシステムリリースです。チームによれば、このスタックは1TパラメータのMoEモデルでのRLをサポートし、ステップ時間は5分未満、約3日で約1,000ステップを達成しています。さらに、GLM-5 agentic SWEセットアップで131kコンテキストを実現しています。リリースでは、推論（wide-EP、FP8 inference、llm-d router、Mooncake、KV-cache CPU offload）、トレーニング（FSDP2、Deep-EP、DSA CP、FP8 training、router replay）、ロールアウトオーケストレーション（コアの書き直し、GLM5、Kimi、Nemotron対応）などの最適化が強調されています。詳細は@PrimeIntellect、技術概要は@samsja19、参考情報は@eliebakouch、@mervenoyannをご覧ください。
  • より広い傾向として、エージェントのトレーニングはアルゴリズムの問題ではなく、インフラの問題になりつつあることが挙げられます。関連する事例として、W&B/OpenPipeはRLスループットをtokens/secではなくtrajectories/secで再定義し、新しいMegatronバックエンドによってARTで12倍のスループットを達成、GRPOのようなワークロードで4GPUで最大35 trajectories/sを実現しています（@wandb）。Vibrant LabsはEcom Benchを公開しました。これは40タスクのライブShopifyベンチマークで、ブラウザエージェントのトレーニングと評価をオープンかつ再現可能にするための決定的検証を備えています（@VibrantLabsAI）。これらは、オープンなトレーニング後スタック＋検証可能な環境＋タスク特化型ロールアウトへの移行を強化しています。

Agent Harness、バックグラウンドエージェント、そして「Async Teammate」UX

  • AnthropicのClaude Tagは、チャットボットから持続的で非同期、組織に埋め込まれたエージェントへの移行を最も明確に示す製品です。ClaudeはSlackにチームメンバーとして参加でき、特定のチャンネルやツールへのアクセス権を持ちます。Anthropicによれば、社内版はすでにプロダクトチームのコードの65%を作成しており、その中にはClaude Tag自体の構築も含まれています。事例は「チャット」用途ではなく、バックグラウンド監視、ローンチや指標のモニタリング、既存ワークフローでのプロアクティブなタスク実行です（@claudeai、@ClaudeDevs、@_catwu）。Andrej KarpathyはこれをLLM UIの第三の主要パラダイムと位置づけています：ウェブサイト→デスクトップアプリ→チームとインラインで働く持続的な存在。
  • オープンエコシステムも同様のアイデアに収束しています。StarAgentはtmux + Tailscale + ウェブダッシュボードを使い、CLIを真実のソースとして保持しつつ、複数のコーディングエージェントセッションをマシン間で多重化します（@ZhihuFrontier）。Self-Harnessは、エージェントが失敗を収集し、ハーネス変更を提案し、回帰テストで検証することを提案しています（@hwchase17）。Hermes Agentは/learnを追加し、ドキュメント、URL、過去のセッションを取り込み、新しいスキルを合成できます（@Teknium）。製品面では、ExecutorがオープンソースのMCPゲートウェイを発表し、エージェントをサービスに接続できるようにしました。セルフホストやデスクトップオプションもあり、現在YC S26に参加しています（@RhysSullivan）。共通テーマは、チームが生のモデルと運用エージェント群の間に欠けているレイヤーを構築しているということです。

オープンモデル、小型モデル、そしてGLM-5.2の勢い

  • 複数の投稿が、特にコーディングやエージェントワークフローにおいて、GLM-5.2を最も注目されたオープンモデルの能力向上として挙げています。 @joshua_saxeは、これほどのレベルのオープンウェイトはAPIログなしで長期的な攻撃的ワークフローを可能にするため、サイバー環境を大きく変えると指摘しています。実務面では、GLM-5.2がフロンティアのクローズドモデルに十分近く、デフォルト選択を変えるという報告が続いています。@_xjdrは、複雑なC++/RustのバグをGPT-5.5 xhighが見逃したのにGLM-5.2が発見したと述べ、@nutlopeは、同等品質で2倍のトークンを生成し、速度は速く、コストはOpusの3分の1だったと報告しています。@UnslothAIは、Mac Studio M3 Ultra 256GBで~21.6 tok/sで動作する1-bit GLM-5.2 GGUFをローカルで実行した事例を示しています。
  • より広く、ルーティング＋小型・低コストモデルがコアスタックのパターンになるという自信が高まっています。 @jpschroederは、DeepSeek V4 FlashがClaude/Codexタスクの約80%を処理でき、Fableよりタスクあたり137倍安価であると述べ、ボトルネックはモデル品質ではなくオーケストレーションだと指摘しています。 @kylebrussellも同様に、チームは「必要なだけの推論」を使い、能力のある小型モデルを活用することを学んでいると述べています。これは、GitHub Copilot AppのBring Your Own KeyのようなBYOK/製品統合アップデートによって強化されています。これはOllama、Foundry、OpenAI互換のcompletions、Anthropic互換のmessage endpointsに対応しています（@_Evan_Boyle）。

インフラと開発者ツール：コンテナ、エンドポイント、カーネルベンチマーク、オブザーバビリティ

  • Appleのcontainerプロジェクトは、MacでDocker Desktopを不要にする有力な手段として注目を集めました。ローカル開発における機能セットは重要で、Apple Silicon上でのLinuxコンテナ、OCI互換性、Swift実装、Apache-2.0ライセンスを備え、Docker Desktopのデーモンや商用ライセンスなしで利用できます（@twtayaan）。
  • 推論インフラでは、ModalがマネージドプライベートLLMエンドポイントを開始し、顧客がブラックボックスサービスではなく基盤コードにアクセスできることを強調しています（@bernhardsson、@akshat_b）。オブザーバビリティでは、Latitudeが繰り返し発生する失敗を課題としてまとめ、実運用会話に対する平易な英語検索、オープンソース/セルフホスト可能な展開で評価されています（@kimmonismus、@omarsar0）。
  • 低レベルの性能作業では、2つの項目が際立ちました。まず、CMUのModern GPU Programming for ML Systems教材がオンラインブックとして公開され、データレイアウトスウィズリング、3D TMA、Blackwellプログラミングなどのテーマを扱っています（@tqchenml）。次に、ParallelKernelBenchは、Megatron-LM、DeepSpeed、DeepEP、TensorRT-LLM、NeMo-RLなどの実際のワークロードからマルチGPUカーネルを作成するLLMの能力をベンチマークしています。現在のフロンティアモデルは依然として苦戦しており、ゼロショットでは87問中28問正解、反復ループを用いても改善は頭打ちで、ランク間の調整や通信メカニズムの推論が難しいことが明らかになっています（@togethercompute、@realDanFu）。

マルチモーダルモデル：OCR、画像モデル、音声、動画

  • Mistral OCR 4は、構造化OCRを提供し、バウンディングボックス、ブロック分類、インライン信頼度スコア、170言語対応を備えています（@MistralAI）。しかし、ベンチマークはすぐに議論となり、@NielsRoggeはMistralのOlmOCRBenchでの「SOTA」主張がHugging Faceの公開リーダーボードと一致せず、現在はオープンモデルの後塵を拝して3位であると指摘しました。一方、BaiduのUnlimited-OCRもHubに登場し、OCR分野が急速に競争的なオープンフロンティアとなっています（@_akhaliq）。
  • 画像生成では、Krea 2が2つのチェックポイントのオープンウェイトを公開しました。Krea 2 Rawは未蒸留の中間トレーニングモデルで、ファインチューニングやトレーニング後に適しています。Krea 2 Turboは高速な蒸留推論モデルです。リリースには技術レポート、day-0 HF/diffusers対応、LoRAエコシステム対応が含まれています（@krea_ai、@fal、@ostrisai）。未蒸留チェックポイントを公開するアプローチは、コミュニティに推論用の完成品だけでなく、実際のトレーニング後の基盤を提供する点で注目されます。
  • 音声と動画では、Artificial AnalysisがSpeech-to-Speech Indexを発表しました。これはBig Bench Audio、Full Duplex Bench、τ-Voiceを組み合わせたもので、総合指標ではGPT-Realtime-2（High）が77.2%でトップ、Grok Voice Think Fast 1.0が75.7%で続き、Geminiのバリエーションがコスト面で競争しています（@ArtificialAnlys）。AssemblyAIは、会話のエージェント側をコンテキストとして利用するリアルタイムASRモデルを導入しました。これは、ボイスエージェントワークフローで、直前にボットが尋ねた内容を知ることで、メールやIDなどの捕捉精度を向上させることを狙っています（@AssemblyAI）。

    • 「出荷中」という主張の多くは実際にはロードマップ項目であり、Kunlun M100の仕様（メモリ容量、帯域幅、TFLOPS）は見つからず、vLLMのサポートは古いKunlunチップに限られています。別のベンダーでは、現在出荷中のC500/C550は64GB（おそらくGDDR6）程度で、144GB HBM3eを備えH200クラスとされるC600は量産待ちであり、投稿は「近日出荷」シリコンに依存しすぎていると批判されています。
