トップニュース：Anthropic Claude Fable 5とMythos 5のリリース

何が起こったのか

Anthropicは次世代モデルファミリーの2つのバージョンを発表しました。一般提供のClaude Fable 5と限定アクセスのClaude Mythos 5です。

  • Anthropicは公式にClaude Fable 5を「初の一般提供Mythosクラスモデル」として発表し、これまで広く提供してきたモデルを上回り、ほぼ全てのテスト済みベンチマークで最先端であると述べました @claudeai, @claudeai
  • Fable 5はMythos 5と同じ基盤モデルですが、安全対策が追加されており、サイバー・バイオ・化学・蒸留関連のプロンプトはClaude Opus 4.8にルーティングされる場合があります @ClaudeDevs, @scaling01
  • Anthropicは「狭い範囲」の潜在的に有害なテーマについてはOpus 4.8へのフォールバックが透明に行われると説明し、初期ユーザー向けメッセージによれば95%以上のセッションでは発生しないとしています @claudeai, @mikeyk
  • フォールバックはサーバー側およびSDKミドルウェア（Python, TypeScript, Go, Java, C#）で利用可能です @ClaudeDevs
  • Fable 5とMythos 5の価格は入力トークン100万あたり$10、出力トークン100万あたり$50と報告され、キャッシュ価格は第三者評価によってキャッシュ書き込み100万あたり$12.50、読み込み100万あたり$1とされています @scaling01, @ArtificialAnlys
  • Fable 5は100万トークンのコンテキストウィンドウを維持しています @ArtificialAnlys
  • Fable 5はPro, Max, Team, Enterprise（席数ベース）プランに6月22日まで含まれ、その後は容量制限のため使用クレジットが必要となり、後に再び広く購読アクセスを提供する予定です @ClaudeDevs, @scaling01, @ArtificialAnlys, @kimmonismus
  • 「6月22日まで含まれる」という表現に混乱が生じ、Anthropicスタッフが展開方法を説明しました @dejavucoder, @TheAmolAvasare
  • 高い需要を受けてAnthropicは後に5時間および週単位のレート制限をリセットしました @ClaudeDevs

公式発表と第三者ベンチマークデータ

Anthropicとパートナーは、特にコーディングや長期的なエージェントタスクで広範なベンチマーク優位性を報告しました。

  • AnthropicはFable 5が特にソフトウェアエンジニアリング、知識業務、科学研究、ビジョンに強く、タスクの長さと複雑さが増すほど優位性が高まると述べています @claudeai
  • CursorはFable 5がCursorBench SOTAで72.9%を記録し、前回最高値を8ポイント上回ったと報告しました @cursor_ai
  • CognitionはFable 5がFrontierCodeで1位となり、Devin Cloud Ultra、Desktop、CLIに統合したと述べています @cognition, @cognition
  • ClineはFable 5がTerminal-Bench 2.1で88.0%を記録し、GPT-5.5を4.6ポイント上回ったと報告しました @cline
  • Artificial AnalysisはFable 5をIntelligence Indexで64.9と評価し、GPT-5.5より約5ポイント高く、Anthropicが上位2位を占めたと述べています @ArtificialAnlys
  • その他詳細なベンチマーク結果やフォールバック率も報告されています。

製品挙動、利用プロファイル、展開詳細

今回のリリースは、評価結果だけでなくワークフローやコスト構造の変化によって特徴づけられました。

  • Fable 5は非常に長く高負荷なタスクに適しており、ユーザーはタスクではなく目標や責任を与える傾向に変化しています。
  • 複数のテスターはFableを「遅く、トークン消費が多く、高価だが非常に有能」と評価しました。
  • StripeがFableを使って5000万行のRuby移行を1日で完了したという事例も紹介されています。
  • Fable 5はすでにCursor, Devin, Notion, Microsoft Foundry, GitHub Copilot App/CLI, Cline, Replit, Base44, MagicPath, Arena, MCP Atlasなどに統合されています。

安全性アーキテクチャと主要な論争

最大の議論はFable/Mythosの強さではなく、Anthropicが一部のフロンティアAI開発タスクで有用性を静かに低減させたことでした。

  • Fable 5がフロンティアLLM開発に使われる場合、Anthropicはプロンプト修正、ステアリングベクトル、PEFTを用いてモデルの効果を制限し、ユーザーには通知しないとしています。
  • サイバーセキュリティやバイオセキュリティ関連のリクエストはOpus 4.8に自動ルーティングされます。
  • 批判者はこれを「有料製品での静かな性能低下は許されない」と主張しました。
  • 一部の研究者はこれをオープン研究やオープンウェイトに対する競争阻害と見ています。

事実と意見

事実 * Fable 5は一般提供、Mythos 5は限定アクセス * 両者は同じ基盤モデルで、Fableには追加の安全策 * 価格は入力100万トークンあたり$10、出力100万トークンあたり$50 * コンテキストは100万トークン * フォールバック機構とSDKミドルウェアを導入 * フロンティアLLM開発への静かな介入は全トラフィックの約0.03%に影響 * Fableは6月22日まで購読プランに含まれる

意見 * 「Anthropicが勝った」「コーディングで優位」などはコメントであり事実ではない * IPO戦略や反オープンソース姿勢などは推測 * 安全性信念からの行動という見方も解釈の一つ

異なる視点

支持的 * Fable 5は大きな進歩であり、長く難しいタスクで特に優れていると評価

批判的 * 静かな性能制限は安全目的でも許されないとする意見が多い

中立的 * 安全のため必要と信じている可能性はあるが、製品設計は不十分との見方

研究制限、プライバシー、企業への影響

  • 企業にとって重要なのは予測可能性であり、静かな性能低下は信頼性を損なう
  • オープンウェイトや社内モデルへの移行を促す可能性がある
  • 再現性や科学的帰属を損なうという懸念もある

コンテキスト

  • GPT-5.5やGemini 3.5 Proとの競争の中でリリース
  • 公開アクセスとドメイン特化の隠れた制限を組み合わせた政策は今後の議論に影響
