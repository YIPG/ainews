Apple、ついに方針転換

AppleのAI部門が苦戦していることに同情する人々は、Siriの約束されたアップデートをまだ受け取っていないAppleユーザーだけでしょう。1年以上にわたり自社開発モデルについて語ってきたAppleは、ついにGoogleのGeminiをSiriの基盤に採用すると発表しました。セキュリティ層は引き続きApple Private Cloud Computeが提供します。これはGoogleにとって大きな勝利であり、比較的OpenAIにとっては痛手です。Appleは当初、AIローンチ時にOpenAIと提携していましたが、今年中に消費者向けハードウェアデバイスを出荷するという噂が高まっており、この決定に影響した可能性があります。

DeepSeekのEngram：条件付きメモリを新たな疎性プリミティブとして

  • Engram = 静的パターン用のO(1)ルックアップ型メモリ：DeepSeekは「Conditional Memory via Scalable Lookup」（Engram）を導入し、ハッシュ化されたn-gram埋め込みメモリをモデルがクエリし、表現にゲートする仕組みを追加しました。従来のトランスフォーマーは、記憶されたパターンを順伝播で再構築するために計算資源を消費しますが、Engramはこの「静的取得」をオフロードし、バックボーンの容量を推論に集中させることができます（@scaling01）。フォローアップでは長文コンテキストにも有効との報告があります（tweet）。
  • システムとスケーリングへの影響：決定的なハッシュ／ルックアップはハードウェア最適化（プリフェッチやメモリ移動）を可能にし、スケーリング制約をHBM依存パラメータから解放します。これにより、FLOPsを増やさずに「知識容量」を安価に拡張できる実用的な道筋が示されました（@tokenbender、@teortaxesTex）。
  • 既存技術との関係（Gemma-3n／N-Grammer／PLE／OTT）：エンジニアはEngramを過去の「n-gram情報を埋め込みに注入する」手法と比較し、Engramはメモリを能動的にレイヤーでアドレスする操作としている点が異なると指摘します（comparison、@arohan）。
  • コミュニティの評価：有望だが「システム志向」で脆弱な可能性もあるとの批評があり、「常時フェッチ＋ゲート」が適切か疑問視する声もあります。予算内での性能向上は現実的だが小幅（約3〜5%）との見方です（@tokenbender）。

長文コンテキスト＆メモリ研究：DroPE、エージェントメモリ、テスト時学習

  • DroPE（Sakana AI）：位置埋め込みを削除してコンテキスト拡張：Sakanaの「DroPE」はRoPEで収束させた後、位置符号化を削除することで長文コンテキスト拡張時の意味歪みを防ぎます。NoPEの学習困難（勾配消失）とRoPEスケーリングによる低周波歪みの中間解として提案されています（paper tweet、Japanese explainer）。
  • テスト時学習を「メモリ」として利用：TTT-E2E：NVIDIA／Stanford／Asteraは「End-to-End Test-Time Training」を提案し、展開時に提供されたコンテキスト上で次トークン学習を継続することで、重要なコンテキストを重みに圧縮し、大規模KVキャッシュへの依存を減らします（NVIDIAAIDev）。
  • エージェントメモリフレームワークの収束：
  • AgeMem：長期＋短期メモリを単一の学習可能ポリシーとして扱い、ツール的な操作（追加／更新／削除＋取得／要約／フィルタ）を段階的RL戦略で訓練します（@omarsar0）。
  • SimpleMem：意味を失わない圧縮＋統合＋クエリ対応取得に注力し、LoCoMoでMem0より高いF1スコアを達成し、クエリあたりのトークン数を大幅に削減します（@dair_ai）。

大手プラットフォームの動き：Apple↔Gemini、Googleの商取引エージェント化、API拡張

  • Apple、Siri／Apple Foundation ModelsにGeminiを採用：共同声明ではGeminiモデルとクラウド技術を基盤にしつつ、オンデバイス＋Private Cloud Computeとプライバシー基準を維持するとしています（statement）。
  • Google、商取引エージェント基盤を推進：Universal Commerce Protocol（UCP）やAI Mode／Geminiでの直接チェックアウト、「Business Agent」チャットなどを発表しました（@Google）。
  • Gemini APIの入力拡張：インラインファイル上限を20MBから100MBに引き上げ、Google Cloud Storageバケットからのネイティブ取り込みや署名付きURLをサポートしました（@osanseviero）。

OpenAIのヘルスケア展開：製品境界、買収、プライバシー方針

  • ChatGPT Health＝専用メモリ領域：健康関連のチャットやファイル、メモリは専用スペースに保存され、通常のチャットには流入しません（announcement）。
  • Torch Healthを買収：検査結果、薬、診察記録を統合するTorchを買収し、ChatGPT Health機能強化を加速します（OpenAI）。

中国AIビジネス＋オープンエコシステムの動向（IPO、採用指標、モデル配布）

  • ZhipuとMiniMaxのIPOストーリーの違い：ZhipuはB2B／B2Gインフラで長い販売サイクルと高いR&Dコスト、MiniMaxは消費者向け／グローバルプラットフォームで成長曲線と改善する利益率という違いがあります（thread）。
  • オープンモデル採用指標の厳密化：「Relative Adoption Metric（RAM Score）」はHugging Faceダウンロード数を時間とサイズで正規化し、1〜9Bモデルが生ダウンロード数で優位だが、上位10モデルの中央値は大規模モデルとの差が約4倍に留まると指摘します（@natolambert）。
  • GLM-4.7の配布＋高速推論：GLM-4.7がCerebrasやTogether AI経由で利用可能になり、200Kコンテキストや高いコーディングベンチマークを謳っています（@NielsRogge）。
