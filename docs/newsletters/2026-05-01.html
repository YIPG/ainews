Grok 4.3のリリース、ベンチマーク差異、そしてオープンvsクローズドの最前線

  • xAIがGrok 4.3をリリース、コスト・性能が大幅改善も評価は賛否両論：@scaling01によるAPIローンチ予告に続き、Artificial Analysisが詳細なベンチマークを公開。Intelligence Indexでは53点を獲得し、Grok 4.20から4ポイント上昇。入力価格は約40%低下、出力価格は60%低下。最大の伸びはGDPval-AAで321 Elo上昇し1500に到達、実世界のエージェントタスク性能向上を示唆。τ²-Bench Telecomで98%、IFBenchで81%を維持。一方でAA-Omniscience精度は上昇したが非幻覚率は8ポイント低下し、能力向上と信頼性低下のトレードオフが懸念される。Arenaはすでにテキスト・ビジョン・ドキュメント・コードモードで対応済み。
  • コミュニティの反応は「意味ある改善」と「依然トップオープンモデルに劣る」で分裂：@teortaxesTexはトークン効率の改善も指摘し、批評家が認める以上に進化していると主張。一方@scaling01は「Grok-4.3は中国のオープンソースに依然劣る」とコメント。Andon LabsはVending-Bench 2で重大な退化を報告、Grokが行動より「睡眠」を選ぶ傾向を示したと指摘。@teortaxesTexは価格が低いのはハードウェア利用効率の悪さによる補助であり、キャッシュ経済がモデル品質以上にエージェントのTCOを決定すると論じた。

DeepSeek V4 Pro、ビジョン／空間推論、そしてオープンウェイトが差を縮める

  • DeepSeek V4 Proは今回のバッチで最も有力なオープンウェイトのコーディング／エージェントモデル：@omarsar0による実践レポートでは、Pi coding agent内でテストした結果、複数ターンのエージェント型コーディングにおいてCodexやClaude Codeに匹敵する初のオープンウェイトモデルと評価。特徴は1Mコンテキスト、ハイブリッドCSA/HCAアテンション設計、KVキャッシュを10%に削減、長コンテキストで推論FLOPsをほぼ4倍削減。セットアップ不要、安定したトレース、Fireworks推論での多段研究／コーディングループが可能。
  • ベンチマーク全体ではオープンウェイトが大幅に接近、ただし最難関タスクでは依然差：Artificial Analysisによると、先週リリースされたトップ3オープンウェイトモデル—Kimi K2.6、MiMo V2.5 Pro、DeepSeek V4 Pro—はIntelligence Indexで52〜54点、対してGemini 3.1 Pro PreviewとClaude Opus 4.7は57、GPT-5.5は60。これらトップオープンモデルはすべて1兆超MoEシステムで、ライセンスは緩やか。Kimiは1T/32B active、MiMoは1T/42B active、DeepSeek V4 Proは1.6T/49B active。残る差はHLE、CritPt、TerminalBench Hard、幻覚率の高いOmniscienceに集中。
  • DeepSeekのマルチモーダル方向性は明示的な空間基盤に集中：@teortaxesTexはARC-AGI-2でDeepSeek-VisionがV4-Proを上回るのは実際の空間推論によると推測。ZhihuFrontierが一時公開後削除した技術レポートでは、推論トレースに直接ボックスやポイントを埋め込み、カウント・迷路解き・経路追跡の「参照ギャップ」を減らすマルチモーダルCoTシステムを説明。スタックはDeepSeek-ViT、CSA圧縮、V4-Flash (284B total / 13B active)を使用。初期テストで弱点は残るものの、視覚推論を単なるテキスト説明ではなく明示的な基盤計算に変える注目すべきアーキテクチャ戦略。

Codexの急速な製品拡張 vs Claude Code、Devin、その他エージェントランタイム

  • Codexは製品速度とUXの完成度で優位：@gdb、@theoらが他製品と比較して高評価。OpenAIはデバイスツールバーを追加し、ブラウザ利用速度を約30%改善（@JamesZmSun）。さらに@reach_vbによるチャット内CIステータス、OpenAIによる設定／プラグイン／エージェントの移行ツール、@OpenAIDevsによる「ペット」機能などを追加。ユーザーは単なるモデルエンドポイントではなく統合環境を提供していると評価。
  • Codex vs Claude CodeはUX＋速度＋テイストのトレードオフ：@theoは「GPT-5.5は賢く詰まりを解消できるが、Opus 4.7は意図やテイストが良いが迷うことがある」と述べ、Claude CodeはTTFT/TPSが遅くツール呼び出しが多い一方、GPT/Codexは直接的で経済的と評価。@scaling01はClaude CodeハーネスでPostTrainBenchにおいてGPT-5.5がOpus 4.7を上回らなかったと指摘。
  • 他のエージェントランタイムも類似のプリミティブに収束：Devinはシェル内ホットキーアクセスを追加（@cognition）、Hermesは監督モデルが完了まで強制する/goalループを追加（@Teknium）、FlueはTypeScriptフレームワークとしてヘッドレス自律エージェントを提供（@FredKSchott）。競争の焦点は生のモデルIQからエージェントハーネス設計へ移行中。

エージェントインフラ：検索、メモリ、HITL、耐久実行

  • 研究の強いシグナルは、エージェントシステムのボトルネックはモデル品質ではなくランタイム設計：@omarsar0によるReaLM-Retrieveは推論中の検索が必要と主張、標準RAGより+10.1% F1、固定間隔IRCoTより47%検索回数減、検索オーバーヘッド3.2倍低減。@dair_aiによるOCR-Memoryは長期軌跡を画像として保存し、損失のない過去コンテンツ取得を可能にし、Mind2WebやAppWorldでSOTA達成。
  • LangChain/LangGraphはマルチユーザー・HITLエージェントのプロダクションプリミティブを強化：@sydneyrunkleはデータ分離、委任認証、運用者RBACをLangSmith Agent Server機能に対応。HITLモードでは人間の返信をツール結果として直接返す機能（tweet）、重要行動や判断保留の耐久的な一時停止／再開機能（tweet）を追加。
  • 耐久実行はスタック全体で一級機能に：Cloudflareはエージェント計画に耐久実行を追加するDynamic Workflowsを発表（@celso）。LangChainはcreate_agentをDeep Agentsの低レベルプリミティブとして位置付け、ファイルシステム、bash、圧縮、フック、サブエージェント拡張に対応（@Vtrivedy10）。エージェントランタイム自体—サンドボックス化、リプレイ、チェックポイント、オーケストレーション—が差別化要因となっている。

ブックマークすべき研究・システム論文

  • 再帰／潜在空間マルチエージェント協調はテキストのみの会話に代わる有力候補に：@omarsar0によるRecursive Multi-Agent Systemsは自然言語ではなく共有潜在再帰計算で通信、平均精度8.3%向上、速度1.2〜2.4倍、トークン削減34.6〜75.6%。
  • Meta FAIRの「自己改善型事前学習」アイデアは今回のバッチで最も重要な学習時論文の一つ：@omarsar0は強力なポストトレーニングモデルが事前学習の接尾辞を安全・高品質に書き換え、RL風事前学習中にモデルロールアウトを評価する手法を紹介。事実性36.2%向上、安全性18.5%向上、生成品質で最大86.3%勝率。
  • Microsoftの合成長期コンピュータ利用環境は有力なデータレシピ：@dair_aiは現実的なファイル・文書を備えた1,000台の合成コンピュータを作成し、平均2,000+ターンの8時間エージェントシミュレーションを実施。コンピュータ利用エージェントのボトルネックはモデル能力だけでなくスケーラブルで現実的な経験データであると指摘。
