OpenAIの音声、Codex、サイバーセキュリティ関連リリース

  • GPT-Realtime-2と新しい音声スタック：OpenAIはAPIでGPT-Realtime-2をリリースしました。これはGPT-5クラスの推論能力を持ち、ツール利用、割り込み対応、長時間会話が可能な最も高性能な音声モデルと説明されています。併せて、GPT-Realtime-Translate（70以上の入力言語／13の出力言語に対応したストリーミング翻訳）や、低遅延ストリーミング文字起こしのGPT-Realtime-Whisperも提供されています。ChatGPTの音声アップデートは今後予定されています。Artificial Analysisによると、GPT-Realtime-2はBig Bench Audioで96.6%、Conversational Dynamicsベンチマークで96.1%を達成し、コンテキスト長を32Kから128Kに拡大、音声価格は据え置きとのことです。Scale AIはAudio MultiChallenge S2SリーダーボードでGPT-Realtime-2を1位に位置付け、指示保持率がGPT-Realtime-1.5の36.7%から70.8% APRに向上したと報告しています。
  • Codexのブラウザ制御機能：OpenAIはmacOSとWindows向けにCodex用Chromeプラグインを提供開始しました。これによりCodexがユーザーのブラウザを乗っ取らずにバックグラウンドタブで動作可能となり、プラグイン利用やログイン済みサイトでのChrome操作、複数ツールを組み合わせたワークフロー（ブラウザフローのデバッグ、ダッシュボード確認、調査、CRM更新など）が可能になります。開発チームはブラウザDevTools、マルチタブ並列処理、Webアプリテストを主要ユースケースとして強調しています。
  • サイバー特化型GPT-5.5アクセス：OpenAIは防御的ワークフロー向けのGPT-5.5 with Trusted Access for Cyberと、限定プレビューのGPT-5.5-Cyberを発表しました。後者は認証強化とアカウント管理下でのレッドチーミング、ペンテスト、検証に利用されます。別途、Micah Carroll氏は、以前のRL実行で偶発的なCoT採点事例をスキャナーで検出したが、それがCoT監視可能性を低下させた明確な証拠は見つからなかったと述べています。

Anthropic、解釈可能性、AI安全ツール

  • Natural Language Autoencoders：AnthropicはNatural Language Autoencodersを導入しました。これはモデルの活性化を人間が読めるテキストに変換し、研究者が疎な特徴や教師ありプローブだけでなく「思考のような」内部表現を検査できるようにする手法です。Miles Brundage氏やML-poweredは、NLAsがプロービングや辞書学習を補完し、計画行動の検出や翻訳バグの特定に役立ったと述べています。Ryan Greenblatt氏は、単一フォワードパスの数学問題では「内部CoT」を復元できなかったと警告し、限界や活性化位置の欠落を示唆しています。
  • Goodfireのニューラルジオメトリアジェンダ：Goodfireはニューラルネットワークが「形で考える」という研究シリーズを開始しました。マニフォールドを行動解釈・制御の基本要素と位置付け、SAE型特徴分断との比較や、学習済みマニフォールドに沿った操作が一貫した世界モデル行動を維持する例を示しています。科学的発見への応用として、科学基盤モデルを逆解析し、曲面マニフォールド内のバイオマーカー構造を発見した事例も紹介しています。
  • Anthropicの安全インフラ：AnthropicはThe Anthropic Instituteの研究アジェンダを公開しました。経済的拡散、脅威とレジリエンス、野生環境でのAIシステム、人間の可視性と制御を伴うAI駆動R&Dに焦点を当てています。また、オープンソースの行動評価ツールPetriをMeridian Labsに移管し、HackerOneでセキュリティバグバウンティを公開しました。

エージェント、RL環境、コーディングワークフロー

  • Hermes Agentの進展：Nous/TekniumはHermes Agent v0.13.0をリリースし、Kanbanによるマルチエージェントオーケストレーション、/goalによる目標達成強制、ディスク使用最適化、カスタムLLMプロバイダ、カスタムゲートウェイチャネルを追加しました。以前のアップデートでは、Hermes Gatewayによるエージェント不要のcronジョブ、--no-skillsによる空プロファイル、Lightpandaを機械ネイティブブラウザバックエンドとして追加しています。
  • CursorのオーケストレーションとPRワークフロー：Cursorは/orchestrateを導入し、プランナー・ワーカー・検証エージェントを再帰的に生成する機能を追加しました。内部的にはスキルのトークン使用を20%削減し、評価を改善、バックエンドのコールドスタート時間を80%短縮しました。Cursor 3では統合PRレビュー機能も追加されています。
  • エージェントインフラパターン：LangGraphはdelta channelsを追加し、長コンテキストエージェントのストレージ膨張を抑えるためチェックポイント履歴を差分保存します。Deep AgentsはDaytona、Modal、Runloop、LangSmithでプロバイダ非依存の隔離実行を可能にするサンドボックスバックエンドを追加し、資格情報をプロンプト注入可能なサンドボックスから守るauth proxyパターンを導入しました。

1. Qwen3.6 27Bローカル推論と量子化

  • Qwen 3.6 27BのMTPによる2.5倍高速推論：llama.cppのMTP対応PRにより、Qwen 3.6 27Bの推論速度が大幅に向上しました。M2 Max 96GBで28 tok/sを達成し、262Kコンテキストに対応。RTX Pro 6000 Max-QではQ8モデルが36 tok/sから78 tok/sに向上しました。視覚機能とMTPの併用はクラッシュするとの警告もあります。
  • Qwen 3.6 27B量子化品質比較：BF16/Q8_0はほぼ正確、Q6_Kは劣化、Q5_K_XL/Q4_K_XL/IQ4_XSは実用的、IQ3_XXSは盤面向きが誤り、Q2_K_XLは構造的に破損。4ビット量子化が実用的な最適点とされ、3ビットも使用可能との意見があります。
  • Qwen3.6 27B uncensored heretic v2 Native MTP Preserved：15のMTPヘッドを保持した非検閲版が公開され、KLD=0.0021、拒否率6/100。MTPヘッドが元の拒否分布を保持している場合、非検閲調整後の出力に影響する可能性が指摘されています。
