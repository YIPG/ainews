Agent Labsはこれだけで十分

AIニュース（2026年1月20日〜1月21日）

私たちは、デカコーン級の資金調達がある場合はタイトルストーリーにするという緩やかなルールを設けています。今回、複数の報道によると、OpenEvidenceが昨年の12倍となる120億ドルの資金調達を行いました。CEOのDaniel Nadler氏はCNBCに対し、「OpenEvidenceは米国の医師の40％に利用され、昨年の年間売上は1億ドルを超えた」と述べています。これは売上の120倍の評価額となります。

AI Twitterまとめ

フロンティアモデルのガバナンス：Anthropicの新しいClaude憲法（CC0）と反応

  • 発表内容：AnthropicはClaudeの望ましい行動や価値観を記した新しい「憲法」を公開し、これを直接トレーニングに使用していると発表しました。重要なのは、この憲法全文がCC0 1.0で公開され、再利用や改変を促している点です（発表、CC0リンク）。Anthropicはこれを社内外の専門家によって形作られる生きた文書と位置付けています（フォローアップ）。
  • コミュニティの反応：Amanda Askell氏は、これは進行中の作業であり、フィードバックを歓迎すると述べました（Askell）。一方で、モデルが「モデルのあるべき姿」を記した文書でトレーニングされるという循環性の「メタ」な奇妙さを指摘する声もありました（scaling01）。また、憲法が「アライメントのシグナル」なのか、実際の害の低減なのか、あるいは広報的な人格行動を組み込んでいるのかという議論もありました（nearcyan、NickEMoran）。
  • エンジニアリング上の影響：Anthropicは、社内のパフォーマンスエンジニアリング課題がOpus 4.5によって解けるようになり、採用評価の再設計を余儀なくされた事例を共有しました。これは「モデルが選考課題に追いつく」具体例です（Anthropic Eng、trishume）。

プロダクション環境でのエージェント：Podiumの「AI従業員」からUX・メモリ・評価まで

  • Podiumの「Jerry」事業ユニット：Podiumは1億ドル以上のAIエージェントARRと1万以上のエージェントを展開していると発表しました。これは中小企業の人員不足（営業時間外のリード対応、電話の取り逃し、離職）を背景に、従来の「ソフトウェア販売」ではなく、製品全体を使いこなすAIオペレーターを販売するという戦略です（Eric Rea）。Tom Loverro氏は、ボードレベルの指標として、燃焼額が9500万ドル→ゼロ、AI ARRがゼロ→1億ドルに約21か月で成長したことを共有し、OpenAIの事例研究を紹介しました（Tom Loverro、Garry Tan）。
  • メモリと長期信頼性がボトルネックに：
    • Agent Cognitive Compressor（ACC）は「コンテキストが多ければ良いエージェントになるわけではない」と主張し、単純な記録再生やナイーブな検索を批判します。ACCはスキーマ制約付きでコミットされた「圧縮認知状態」を維持し、長期運用でのドリフトや幻覚を減らすとしています（dair_ai）。
    • 別のスレッドでは、物理学の曖昧なプロンプトを明確化し、コード生成・実行・エラー診断・多言語説明を行うMCP-SIMという自己改善型マルチエージェントループを紹介。ワンショットGPTが6/12しか解けないタスクを12/12解決したとしています（omarsar0）。
  • エージェント評価はコーディングパズルを超える：
    • APEX-AgentsはGoogle Workspaceでの長期的な「プロフェッショナルサービス」タスクを評価。初期のPass@1リーダーボードは低く、Gemini 3 Flash Highが24.0%、GPT-5.2 Highが23.0%、Claude Opus 4.5 Highが18.4%と、エージェントの自律性がまだ脆いことを示しています（BrendanFoody）。
    • prinzbenchは法律調査＋検索のプライベートベンチマーク（33問、手動採点、3回実施）を導入。「検索」が失敗要因であり、GPT-5.2 Thinkingがわずかに50%超、Geminiモデルが僅差で続き、Sonnet/Opus 4.5はSearchで0/24だったとしています（deredleritt3r）。
  • ツールとUX層の進化：複数の投稿が、エージェントにはモデル改善だけでなく「コンテキスト層」や本番運用のための足場（ガバナンス、認証、可観測性）が必要だと指摘しています。

エージェントプラットフォームと「コンテキスト層」：MCP、Skills、Prefect Horizon、LangChain Deep Agents

  • Prefect Horizon：MCPからプラットフォームへ：Prefectは「コンテキスト層」をエージェントと企業のツール・データのインターフェースと位置付け、MCPはサーバー構築方法を記述するが、組織規模での展開・ガバナンス方法は示していないと指摘。Horizonは管理されたデプロイ、レジストリ／カタログ、RBAC＋監査ログ付きゲートウェイ、ビジネスユーザー向けエージェントインターフェースを提供するとしています（jlowin）。
  • MCPサーバー設計ガイド：Phil Schmid氏は「SkillsがMCPを置き換える」という見方に反論。問題はMCPではなく悪いサーバーであり、成果に基づいたツール設計、制約付き型引数、docstringやエラーをエージェント指示として活用することを推奨。SkillsとMCPは補完関係にあるとしています（philschmid）。
  • LangChain Deep Agents：フォルダとしてのエージェント＋UI統合：
    • CopilotKitは、履歴書取り込み→スキル抽出→ウェブ検索付きサブエージェント→ストリーミングUIというフルスタックDeep Agentアプリのチュートリアルを公開（CopilotKit）。
    • LangChainはAgent Builder GAと、Tavily、PagerDuty、Boxなどのドメインパートナーとのテンプレートライブラリを提供し、「プロンプトからエージェントへ」の摩擦を減らしています（LangChain）。
    • Deep Agentsの「エージェントは単なるフォルダ」という考え方は、CLIフローで迅速にパッケージ化・ダウンロード・実行できる移植性／配布性を強調（hwchase17、Vtrivedy10、LangChain_OSS）。Sydney Runkle氏は、コンテキスト分離のためのサブエージェントと必要時のみロードされるスキルの2つのパターンを強調しています（sydneyrunkle）。
  • LangSmith＋分析：LangSmithのトレースはデバッグだけでなくプロダクト分析の基盤にもなり得るという指摘があります（SoftwareWatcher）。
